# 用户偏好

## 工作方式
- 用户倾向于在一个会话中持续开发，定期保存进度
- 喜欢使用 /remember 保存重要上下文

## 反馈指导
- 用户希望看到简洁的进度总结
- 代码问题直接修复，不要过多解释
- 文档更新要及时
- 问题未解决时会持续追问直到彻底解决
- 遇到问题希望立即看到修复，不要让用户手动执行临时方案

## 项目规范
- 详细设计文档放在 `04-功能详细设计/` 目录，打时间标签
- 需求文档放在 `02-需求和项目介绍/` 目录
- 进度更新放在 `08-项目进度/` 目录
