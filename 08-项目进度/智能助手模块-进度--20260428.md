## 一、模块概述

**模块名称**：智能助手（Intelligent Assistant）

包含四个子模块：
1. **问答助手** (`/qa/chat`) - 通用知识问答，基于 RAG
2. **执法智能助手** (`/assistant/law`) - 执法场景处置推荐
3. **工程监管助手** (`/assistant/supervise`) - 工程质量安全问题定位
4. **图谱增强问答** (`/assistant/graph`) - 基于知识图谱的多跳推理问答

## 三、本次完成内容（2026-04-28）

### 3.1 图谱上下文增强（核心重构）

**问题**：Neo4j 图谱数据只用于前端可视化，没有参与 LLM 回答生成。

**解决方案**：新增 `_build_graph_context` 方法，将图谱关系加入 LLM prompt。

#### 修改的方法

| 方法 | 改动 |
|------|------|
| `_generate_answer` | 新增 `graph_context` 参数 |
| `_query_neo4j_context` | 匹配时尝试书名号变体 |
| `_build_graph_context` | 新增方法 |
| `_match_neo4j_entity` | 新增方法（处理书名号） |
| `_neo4j_entity_exists` | 新增方法 |

#### 最终传给 LLM 的数据结构

```
用户问题：{question}

知识内容：
【知识 1】{title}
{content[:500]}
...

=== 知识图谱关系（供参考）===

【实体分类】
  法规：《安全生产许可证条例》, 《建筑法》, ...
  条款：第一条, 第二条, 第三条
  主体：住房公积金所有者

【关系链条】
  • 《安全生产许可证条例》 包含 第一条
  • 第一条 包含 《建筑法》
  • ...

（以上图谱包含 69 个实体和 97 条关系）
```

### 3.2 书名号匹配修复

处理实体名称中书名号的差异：
- `"安全生产许可证条例"` → 匹配 → `"《安全生产许可证条例》"`
- `"《建筑法》"` → 尝试匹配 → `"建筑法"`
