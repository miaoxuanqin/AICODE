#### 问题发现

- user_id 类型不匹配（字符串 vs 整数）

#### 修复内容

| 文件 | 修改 |
|-----|------|
| `app/api/v1/knowledge.py` | upload/manual 接口添加 index_vector() 调用 |
| `app/services/search_service.py` | search_vector() 修复 API 调用和类型处理 |
| `app/scripts/rebuild_vector_index.py` | 新增批量重建索引脚本 |

#### 修复后流程
```
POST /knowledge/upload → index_knowledge (ES) + index_vector (Qdrant)
```

### 4. 批量重建索引脚本

```python
# app/scripts/rebuild_vector_index.py
# 用法：
cd code/backend
python -m app.scripts.rebuild_vector_index --recreate
```

**功能：**
- 删除旧 collection（解决维度不匹配问题）

- 批量为所有知识建立向量索引

## 搜索效果演示

### 语义搜索 vs 关键词搜索

| 搜索词 | 关键词搜第1条 | 语义搜第1条 | 差异说明 |
|-------|-------------|------------|---------|
| 高处作业 | 施工现场高处作业安全管理规定 | 施工现场高处作业安全管理规定 | 相同 |
| 完工 | 建筑工程竣工验收统一标准 | 建设工程竣工验收备案流程指南 | 不同！ |
| 处罚 | 某建筑公司违反安全生产规定处罚案例 | 某项目违规分包查处案例分析 | 不同！ |
| 小区私搭乱建 | 某小区违法建设强制拆除案例 | 某小区违规装修查处案例 | 不同！ |

### 关键发现

**"完工" 搜索最能体现语义搜索价值：**
- 知识库中写的是"竣工验收"而不是"完工"
- 关键词搜索能匹配（因为摘要/内容中有"验收"）
- 语义搜索能理解"完工 ≈ 竣工"，返回不同排序
