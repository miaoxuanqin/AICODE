## 技术方案

### 会话管理策略
- **独立会话** - 各助手维护自己的会话（通过 category 字段隔离）
- **静默管理** - 用户无感知，不显示会话列表
- **自动关联** - 进入页面时自动创建 category=law_general 的会话
- **会话隔离** - 执法助手会话通过 category='law_general' 标识，与问答助手会话分开

### Session Category 设计
| category | 用途 | 可见页面 |
|----------|------|----------|
| qa | 问答助手会话 | QAChat.vue |
| law_general | 执法助手会话 | LawAssistant.vue |
| supervise | 工程监管助手会话 | SuperviseAssistant.vue（待开发） |

### 架构
```
执法助手 → qaApi.chat() → qa_service (RAG) → ES搜索 + MiniMax LLM
```

### 1. LawAssistant.vue 后端对接 ✅ 已完成

- 创建独立会话 (category='law_general')

### 2. 会话隔离问题修复 ✅ 已完成
- **问题**：执法助手和问答助手共用 Session 表，会话列表混在一起
- **解决方案**：QASession 表添加 category 字段区分不同助手
- **实现**：
  - 后端：QASession 模型添加 category 字段
  - API：createSession 支持 title 和 category 参数
  - 前端：LawAssistant 创建会话时传入 category='law_general'
  - 前端：QAChat 加载会话时过滤掉 category='law_general'

### 3. 思考中提示位置修复 ✅ 已完成
- **问题**："正在检索知识库并生成回答，请稍候..." 显示在消息列表最上面
- **解决方案**：将思考中提示 DOM 移到消息循环之后
- **影响页面**：LawAssistant.vue、QAChat.vue

### 4. 关联知识动态加载 ✅ 已完成
- **功能**：用户发送消息后，基于场景描述调用 `knowledgeApi.search` 动态搜索
- **显示**：右侧卡片展示相关知识（标题、分类标签）
- **交互**：点击跳转到知识详情页
- **状态**：加载中显示 loading 图标，无数据显示"暂无关联知识"

### 5. 最近案例动态加载 ✅ 已完成
- **功能**：复用执法助手自己的会话历史，取用户的问题作为案例
- **显示**：右侧卡片展示最近 5 条案例记录
- **交互**：点击将案例内容填充到输入框，可继续追问

### 6. Markdown 答案渲染 ✅ 已完成
- **问题**：LLM 返回 Markdown 格式答案，前端直接渲染导致格式标记显示为普通文本
- **解决方案**：安装 marked 库，使用 v-html + marked 渲染 Markdown 答案
- **影响页面**：LawAssistant.vue、QAChat.vue
