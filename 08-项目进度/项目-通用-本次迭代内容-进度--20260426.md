## 二、问答助手消息顺序修复

### 问题

快速发送多条消息时，由于异步请求返回顺序不确定，消息显示顺序错乱。

### 解决方案

1. 添加 `currentRequestId` 追踪当前请求
2. 每条消息添加 `timestamp` 时间戳
3. 使用 `sortedMessages` computed 属性按时间戳排序
4. 请求响应时检查是否是最新请求，避免旧请求覆盖

### 修改文件

- `src/views/qa/QAChat.vue` - 添加请求追踪和消息排序

## 三、执法智能助手初始化逻辑修复

### 问题

进入执法智能助手页面时，关联知识、最近案例、会话框都是空的。

### 原因

1. 进入页面创建新会话后，没有加载已有会话历史
2. 关联知识和最近案例只在发送消息后才加载

### 解决方案

修改 `initSession` 逻辑：
1. 先获取已有的执法助手会话（category='law_general'）
2. 如果有历史会话，加载会话记录、关联知识、最近案例
3. 如果是新会话，则创建新会话

新增 `loadSessionHistory` 方法，统一加载会话历史和相关数据。

### 修改文件

- `src/views/assistant/LawAssistant.vue` - 重构 initSession 和新增 loadSessionHistory

## 五、普通用户权限控制

### 问题 1：403 错误

普通用户登录后访问用户管理、角色管理等页面时，出现 403 错误。

### 原因

前端路由虽然配置了 `requireAdmin: true`，但普通用户仍能通过直接输入 URL 访问，系统管理功能调用后端 API 时返回 403。

### 解决方案

1. **路由守卫**（router/index.js）：
   - 检查路由 meta 中的 `requireAdmin` 标志
   - 非 admin 用户访问管理页面时，提示"您没有访问该页面的权限"并跳转回知识搜索页

2. **登录跳转逻辑**（Login.vue）：
   - admin 用户登录后跳转到 `/system/users`
   - 普通用户登录后跳转到 `/knowledge/search`

### 修改文件

| 文件 | 说明 |
|------|------|
| `src/router/index.js` | 新增路由守卫检查 requireAdmin |
| `src/views/auth/Login.vue` | 修复登录后根据角色跳转 |

### 问题 2：用户列表/角色列表 403 错误重复提示

前端多个页面调用获取用户列表/角色列表 API 时，403 错误被重复提示。

### 解决方案

在 UserManagement.vue 和 RoleManagement.vue 的 fetch 函数中添加 403 状态码过滤：

```javascript
if (error.response?.status !== 403) {
  console.error('获取用户列表失败:', error)
  ElMessage.error('获取用户列表失败')
}
```

- `src/views/system/UserManagement.vue`
- `src/views/system/RoleManagement.vue`
