## 三、向量搜索 vs 关键词搜索对比演示

### 测试结果

**问题：** `楼板出现裂缝怎么处置？`

| 搜索类型 | 结果 | 说明 |
|----------|------|------|
| **向量搜索** | 违规装修查处案例、钢筋配置不规范 | 语义理解"楼板裂缝"=建筑质量问题 |
| **关键词搜索** | 安全生产相关规定 | 字面匹配"楼板"/"安全" |

**问题：** `混凝土强度不足怎么处理？`

| 搜索类型 | 结果 | 说明 |
|----------|------|------|
| **向量搜索** | 混凝土强度不足的处理措施 ✅ | 精确语义匹配 |
| **关键词搜索** | 混凝土强度不足的处理措施 ✅ | 精确关键词匹配 |

### 差异分析

1. **向量搜索**：语义理解能力强，同义词/相关概念能关联
2. **关键词搜索**：精确匹配，字面不符则搜不到

### 演示命令

```python
from app.services.embedding_service import embedding_service
from app.services.search_service import search_service

# 向量搜索
query_vector = embedding_service.encode_single("楼板裂缝怎么办")
vec_results = search_service.qdrant.query_points(
    collection_name='knowledge',
    query=query_vector,
    limit=3
)
for hit in vec_results.points:
    print(hit.payload.get('title'))

# 关键词搜索
kw_results = search_service.search_keyword(
    query="楼板裂缝怎么办",
    user_id='1',
    page=1,
    page_size=3
)
for item in kw_results['items']:
    print(item['title'])
```

## 四、终端编码问题说明

### 问题现象

Windows 终端输出中文时显示乱码：
```
============================================================
��������: ���蹤���������ձ�׼��ʲô��
```

### 原因分析

Windows 终端默认使用 GBK/CP936 编码，Python 输出 UTF-8 编码时显示乱码。

### 解决方案

在 Python 脚本中强制设置 UTF-8 输出：
```python
import sys
import io
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8')
```

### 验证结论

终端乱码是**显示问题**，实际数据/代码正常。数据库中中文存储正确。
