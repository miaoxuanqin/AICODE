## 一、模块概述

**模块名称**：图谱增强问答（Graph-RAG）

**功能定位**：基于知识图谱的多跳推理问答系统，在传统 RAG 基础上引入实体关系图谱，增强回答的可解释性和完整性。

**与现有助手的关系**：

| 助手 | 定位 | 技术 | 特点 |
|------|------|------|------|
| 问答助手 | 通用知识问答 | RAG | 单跳检索 |
| 执法智能助手 | 执法场景推荐 | RAG + 模板 | 领域化 |
| 工程监管助手 | 质量问题定位 | RAG + 向量 | 语义搜索 |
| **图谱增强问答** | **多跳推理** | **Graph-RAG** | **可解释推理路径** |

## 三、本次完成内容（2026-04-27 晚）

### 1. Bug修复 - 图谱节点乱跳

**问题**：鼠标悬停在图谱节点上时，节点位置跳动

**原因分析**：
1. `hoveredNode` 存储整个响应式对象，触发 Vue 重渲染
2. CSS `transform: scale(1.1)` 与 SVG `translate()` 变换冲突

**修复方案**：
- 将 `hoveredNode` 改为 `hoveredNodeId`，存储 ID 而非对象
- 移除 `.node-group:hover` 的 `transform: scale(1.1)` 样式
- 悬停效果改用 `:r` 属性控制（35 → 38px）

**修改文件**：
- `code/frontend/src/views/graph/GraphQAChat.vue`

### 2. Feature - 推理步骤详情展开

**功能**：点击推理步骤，展开查看该步骤的搜索详情

**实现方案**：
- 后端：每个推理步骤增加 `details` 字段
  ```python
  "details": {
    "search_query": "搜索词",
    "knowledge_found": ["知识1", "知识2", ...]
  }
  ```
- 前端：添加 `expandedSteps` 状态和 `toggleStep` 方法
- 点击步骤行或箭头图标触发展开/收起

**修改文件**：
- `code/backend/app/services/graph_service.py` - 添加 details 字段
- `code/backend/app/api/v1/graph.py` - ReasoningStep 模型添加 details 字段
- `code/frontend/src/views/graph/GraphQAChat.vue` - 展开功能实现

### 3. Bug修复 - 推理详情不显示

**问题**：后端返回的 reasoning_chain 没有 details 字段

**原因**：Pydantic 模型 `ReasoningStep` 没有定义 `details` 字段，被自动过滤

**修复**：
```python
class ReasoningStep(BaseModel):
    step: int
    query: str
    result: str
    entities: Optional[List[str]] = []
    details: Optional[Dict[str, Any]] = {}  # 新增
```

### 4. Bug修复 - 图谱问答 404

**问题**：调用 /api/v1/graph/qa 返回 404

**原因**：Python `__pycache__` 缓存了旧字节码

**修复**：
```bash
find . -type d -name __pycache__ -exec rm -rf {} +
taskkill //F //IM python.exe
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## 五、技术方案

### 5.1 当前实现（基于知识库）

```
用户问题 → 实体识别 → 知识检索 → 关系扩展 → 图谱构建 → 回答生成
                                    ↓
                              可视化图谱展示
```

### 5.2 推理步骤详情

每个推理步骤现在包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| step | 步骤编号 | 1, 2, 3, 4 |
| query | 步骤名称 | "理解问题并识别关键实体" |
| result | 结果描述 | "识别到 5 个关键实体" |
| entities | 实体列表 | ["脚手架", "坠落"] |
| details.search_query | 实际搜索词 | "脚手架坠落事故如何处罚" |
| details.knowledge_found | 找到的知识 | ["建筑施工特种作业人员管理规定", ...] |

### 5.3 复用组件

| 组件 | 复用方式 |
|------|----------|
| embedding_service | 向量编码 |
| search_service | ES + Qdrant 搜索 |
| LLM (MiniMax) | 回答生成、实体抽取辅助 |
