## 一、模块概述

**模块名称**：图谱增强问答（Graph-RAG）

**功能定位**：基于知识图谱的多跳推理问答系统，在传统 RAG 基础上引入实体关系图谱，增强回答的可解释性和完整性。

**技术架构**：
- 当前：向量检索 + 内存图谱 + 前端动态生成
- 已完成：Neo4j图数据库集成（待用户部署环境）

## 三、本次完成内容（2026-04-28）

### 3.1 前端Neo4j开关集成

**问题**：后端已接入Neo4j但前端未启用

**修复**：
1. `graphApi.qa()` 添加 `useNeo4j` 参数
2. GraphQAChat.vue 添加 Neo4j 开关
3. 调用时传入 `useNeo4j.value`

### 3.2 思考中界面优化

**原问题**：等待答案时只显示简单进度条，用户体验单调

**优化**：
- 实时推理链展示（4个步骤逐步展开）
- 进度百分比动态更新
- 步骤状态动画（等待→进行中→完成）
- 连接线渐变效果
- 实体标签实时显示
- "正在分析..."动态点动画

### 3.3 趣味提示语

```javascript
const funTips = [
  '正在知识的海洋中遨游...',
  '让AI仔细思考一下这个问题...',
  '正在建立知识点之间的联系...',
  '这个问题有点意思，让我分析分析...',
  '调用脑细胞中...',
  '知识图谱构建ing...',
  '答案马上就到～',
  '正在调动专业知识库...'
]
```

### 3.4 图谱动态效果

**节点动画** (`nodeAppear`)：
- 缩放弹跳效果 (scale 0.3 → 1.1 → 1)
- 淡入效果

**边动画** (`edgeDraw`)：
- 线条绘制动画 (stroke-dashoffset)
- 绘制完成后正常显示

**SVG transition-group**：
- 包裹节点和边，支持Vue过渡动画

**数据监听**：
- watch监听graphData变化
- 检测到新节点时自动添加动画class

### 3.5 动态图谱生成

**背景**：后端返回空图谱时，前端根据问题内容动态生成

**生成逻辑**：
1. 提取reasoningChain中的实体和知识
2. 问题节点作为中心
3. 知识节点围绕问题节点分布
4. 实体节点作为关联补充
