## 一、模块概述

**模块名称**：图谱增强问答（Graph-RAG）

**功能定位**：基于知识图谱的多跳推理问答系统，在传统 RAG 基础上引入实体关系图谱，增强回答的可解释性和完整性。

**技术架构**：
- 当前：向量检索 + 内存图谱
- 规划：向量检索 + Neo4j 图数据库

## 三、本次完成内容（2026-04-28）

### Neo4j 集成开发

#### 1. Neo4j 服务（neo4j_service.py）

**新增功能**：
- 节点 CRUD：upsert_node, get_node, delete_node
- 关系 CRUD：create_relation, delete_relation
- 图谱查询：get_related_entities, find_paths, get_subgraph
- 引用追踪：get_entities_by_knowledge_id, get_node_ref_count
- 智能清理：remove_knowledge_reference

**核心方法**：
```python
upsert_node(label, name, properties, knowledge_id)
# 插入/更新节点，自动建立 EXTRACTED_FROM 关系

remove_knowledge_reference(knowledge_id)
# 移除知识引用，无引用时删除实体
```

#### 2. 实体关系抽取（graph_extractor.py）

**抽取策略**：
- 规则快速抽取：正则匹配法规、条款、标准、处罚等
- LLM 语义抽取：理解实体类型和关系
- 合并去重：规则 + LLM 结果合并

**核心方法**：
```python
extract(knowledge_id, text)
# 执行抽取，返回 entities 和 relations

sync_to_neo4j(knowledge_id, text)
# 抽取并同步到 Neo4j

delete_from_neo4j(knowledge_id)
# 删除时清理 Neo4j
```

#### 3. 知识同步钩子（knowledge.py）

**入库时同步**：
```python
# 异步同步到 Neo4j（图谱抽取，不阻塞主流程）
if parsed_content:
    asyncio.create_task(
        asyncio.to_thread(_sync_knowledge_to_neo4j, knowledge_id, parsed_content)
    )
```

**删除时清理**：
```python
# 删除 Neo4j 相关数据
extractor.delete_from_neo4j(knowledge_id)
```

#### 4. Graph API 扩展

**新增接口**：
- GET /neo4j/status - Neo4j 连接状态

## 四、删除逻辑

### 4.1 EXTRACTED_FROM 关系

每个实体通过 `EXTRACTED_FROM` 关系关联到来源知识：

```
实体 -[EXTRACTED_FROM]-> 知识A
实体 -[EXTRACTED_FROM]-> 知识B  (同一实体被多个知识引用)
```

### 4.2 删除流程

```
知识删除 → 移除 EXTRACTED_FROM 关系 → 检查 ref_count
                                    ├── ref_count == 0 → 删除实体
                                    └── ref_count > 0 → 保留实体
```

### 4.3 示例

```
实体"噪声污染防治法"被 知识A 和 知识B 引用

删除知识A时：
- 移除 A-[:EXTRACTED_FROM]->实体
- ref_count = 1 (还有B引用)
- 实体保留

删除知识B时：
- 移除 B-[:EXTRACTED_FROM]->实体
- ref_count = 0 (无任何引用)
- 删除实体及其所有关系
```
