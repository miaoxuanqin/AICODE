### 1.1 PDF 预览功能修复

**问题现象**：点击 PDF 文件预览时，一直显示"正在加载PDF..."，没有任何反应。

**根本原因**：
1. PDF.js worker 加载方式不正确，导致 PDF 解析无法完成
2. `pdf-preview-container` 容器在 `previewItem.loading=true` 时被 v-if 隐藏，导致 DOM 查询不到
3. Axios 响应拦截器返回的是整个响应对象 `{data: ..., status: ...}`，需要从中提取 `data`

**修复方案**：
1. 改用动态 URL 构造加载 worker：
```javascript
const setupPdfWorker = () => {
  if (!pdfWorkerSrc) {
    pdfWorkerSrc = new URL('pdfjs-dist/build/pdf.worker.min.mjs', import.meta.url).href
    pdfjsLib.GlobalWorkerOptions.workerSrc = pdfWorkerSrc
  }
}
```

2. 等待容器出现后再渲染：
```javascript
let container = document.getElementById('pdf-preview-container')
while (!container && waitCount < 100) {
  await new Promise(r => setTimeout(r, 100))
  container = document.getElementById('pdf-preview-container')
}
```

3. 从 Axios 响应中提取实际数据：
```javascript
const rawData = response.data || response
const blob = new Blob([rawData], { type: 'application/pdf' })
```

### 1.2 文本类型知识预览

**问题现象**：文本类型（file_type=html/unknown）的知识预览看不到内容。

**原因**：文本类型知识不走文件下载路线，但原来的代码只在 `hasFile=true` 时处理预览。

**修复方案**：
- 无文件的知识（文本类型）直接从 API 获取内容：
```javascript
previewItem.value = { id, title, file_type, ... }
showPreview.value = true
const detail = await knowledgeApi.get(item.id)
previewItem.value.content = detail.content || ''
```

### 1.3 下载按钮显示控制

**需求**：文本类型预览界面不需要下载按钮，只有 PDF/Word 文件需要。

**修复**：
- PDF/Word 预览底部显示下载按钮
- 文本类型只显示内容区域，无下载按钮

### 1.4 图谱重建接口修复

**问题现象**：图谱重建返回 `{status: 'failed'}`，提示"已提交重建任务"但实际失败。

**根本原因**：
- 后端 `rebuild` 接口试图从 `knowledge.content` 获取内容
- 但 MySQL 的 knowledge 表不存 content，只存 ES
- 对于没有文件的知识（文本类型），content 从未写入 ES

**修复方案**：
```python
# 从ES获取content（MySQL不存完整content）
content = None
try:
    es_doc = search_service.get_by_id(knowledge_id)
    if es_doc:
        content = es_doc.get("content")
except Exception:
    pass

if GRAPH_EXTRACTOR_AVAILABLE and content:
    # 执行图谱重建
```

## 二、技术细节

### 2.1 文件类型判断逻辑

```javascript
// 有 file_path 的是文件类型知识，走下载预览
if (item.file_type && item.file_path) {
    // PDF/Word 文件预览
} else {
    // 文本类型，直接从API获取内容
}
```

### 2.2 预览组件结构

```html
<el-dialog v-model="showPreview" title="知识预览" width="900px" top="5vh">
  <div class="preview-header">标题 + 类型标签</div>
  <div class="preview-content">
    <!-- 文本内容显示 -->
    <div v-if="!previewItem.file_type || previewItem.file_type === 'html' || previewItem.file_type === 'unknown'" class="text-preview" v-html="previewItem.content"></div>
    <!-- PDF预览容器 -->
    <div v-if="previewItem.file_type === 'pdf'" id="pdf-preview-container"></div>
    <div v-if="previewItem.file_type === 'pdf' && previewItem.loading" class="preview-loading">正在加载PDF...</div>
    <!-- Word预览 -->
    <div v-if="(previewItem.file_type === 'docx' || previewItem.file_type === 'doc') && !previewItem.loading" class="word-preview" v-html="previewItem.wordContent"></div>
    <!-- 文件下载按钮 -->
    <div v-if="previewItem.file_type === 'pdf' || previewItem.file_type === 'docx' || previewItem.file_type === 'doc'" class="preview-toolbar">
      <el-button @click="downloadFile">下载</el-button>
    </div>
  </div>
</el-dialog>
```

### 2.3 PDF 渲染流程

1. 调用 `knowledgeApi.download(id)` 获取文件 Blob
2. 创建 `Blob([response.data], {type: 'application/pdf'})`
3. 调用 `renderPdfPreview(blob)`
4. 等待容器出现（最多10秒）
5. `pdfjsLib.getDocument({data: arrayBuffer})` 创建 PDF 文档
6. 逐页渲染到 canvas

### 2.4 Word 渲染流程

1. 获取文件 Blob
2. `mammoth.convertToHtml({arrayBuffer})` 转换为 HTML
3. 将 HTML 插入 `.word-preview` 容器

### 4.1 图谱重建失败

对于 `file_type=unknown` 且 `file_path=null` 的知识，图谱重建会返回 `failed`，因为：
- 这类知识可能是手动创建的文本知识
- 手动创建时 content 写入了 ES
- 但如果创建时索引失败，ES 里也没有 content

**解决方案**：需要确保文本知识创建时 content 正确写入 ES。
