## 一、项目概述

### 1.1 项目名称

个人中心

### 1.2 项目定位

为知识库系统用户提供统一的个人数据管理入口，集中管理用户的收藏知识、历史评论和上传内容，提升用户体验。

### 1.3 项目背景

用户在收藏知识或发表评论后，缺乏统一的入口查看这些内容。需要在知识库系统中构建个人中心模块，提供：
- 我的收藏：查看用户收藏的知识列表
- 我的评论：查看用户发表的评论及对应的知识
- 我的上传：查看用户上传的知识文档

## 二、功能需求

### 2.1 功能清单

| 序号 | 功能点 | 功能描述 | 优先级 |
|------|--------|----------|--------|
| F01 | 个人资料展示 | 展示用户头像、用户名、角色等信息 | P0 |
| F02 | 我的收藏 | 列表展示用户收藏的知识，支持取消收藏 | P0 |
| F03 | 我的评论 | 列表展示用户发表的评论，含知识标题链接 | P0 |
| F04 | 我的上传 | 列表展示用户上传的知识文档 | P1 |
| F05 | Tab 切换 | 支持收藏/评论/上传 Tab 切换 | P0 |
| F06 | 分页加载 | 支持列表分页加载 | P1 |

### 2.2 详细需求说明

#### F01: 个人资料展示

**需求描述**：在页面顶部展示当前登录用户的基本信息

**展示内容**：
| 字段 | 说明 |
|------|------|
| 用户名 | current_user.username |
| 姓名 | current_user.full_name |
| 角色 | current_user.role_name |
| 组织 | current_user.org_name |
| 注册时间 | current_user.created_at |

**布局**：横向卡片展示，居左或居中

#### F02: 我的收藏

**需求描述**：展示用户收藏的知识列表

**数据来源**：
- API: `GET /knowledge/favorites` 或 `GET /users/{id}/favorites`
- 返回：知识列表（id, title, category, summary, created_at）

**功能要求**：
| 功能 | 说明 |
|------|------|
| 列表展示 | 卡片式或列表式展示 |
| 取消收藏 | 每行有取消收藏按钮 |
| 跳转到详情 | 点击标题跳转到知识详情页 |
| 空状态 | 无收藏时显示"暂无收藏" |

**字段映射**：
| 字段 | 说明 |
|------|------|
| title | 知识标题 |
| category_name | 分类名称 |
| summary | 摘要（截取前100字） |
| created_at | 收藏时间 |

#### F03: 我的评论

**需求描述**：展示用户发表的评论列表

**数据来源**：
- API: `GET /users/{id}/comments` 或 `GET /knowledge/comments?user_id={id}`
- 返回：评论列表（id, knowledge_id, knowledge_title, content, created_at）

**功能要求**：
| 功能 | 说明 |
|------|------|
| 列表展示 | 按时间倒序展示 |
| 跳转到知识 | 点击知识标题跳转到知识详情页并定位到评论区 |
| 删除评论 | 可删除自己的评论 |
| 空状态 | 无评论时显示"暂无评论" |

**字段映射**：
| 字段 | 说明 |
|------|------|
| knowledge_title | 所属知识标题（点击跳转） |
| content | 评论内容 |
| created_at | 评论时间 |

#### F04: 我的上传

**需求描述**：展示用户上传的知识文档

**数据来源**：
- API: `GET /knowledge?user_id={id}&page=1&page_size=10`
- 返回：知识列表（id, title, category, file_type, status, created_at）

**功能要求**：
| 功能 | 说明 |
|------|------|
| 列表展示 | 展示上传的知识 |
| 状态标识 | 显示索引状态（es_indexed, vector_indexed, graph_indexed） |
| 跳转到详情 | 点击标题跳转到知识详情页 |
| 空状态 | 无上传时显示"暂无上传" |

**字段映射**：
| 字段 | 说明 |
|------|------|
| title | 知识标题 |
| category_name | 分类名称 |
| file_type | 文件类型（PDF/DOC/TEXT） |
| status | 处理状态 |
| created_at | 上传时间 |

#### F05: Tab 切换

**需求描述**：通过 Tab 页签切换不同模块

**Tab 页签**：
| Tab | 内容 |
|-----|------|
| 我的收藏 | 收藏列表 |
| 我的评论 | 评论列表 |
| 我的上传 | 上传列表 |

**交互要求**：
- 默认选中"我的收藏"
- Tab 切换时重新加载数据
- 支持键盘快捷键切换

#### F06: 分页加载

**需求描述**：列表数据支持分页

**分页配置**：
| 配置 | 值 |
|------|------|
| 默认每页 | 10条 |
| 每页可选 | 10/20/50 |
| 最大总数 | 不限 |
