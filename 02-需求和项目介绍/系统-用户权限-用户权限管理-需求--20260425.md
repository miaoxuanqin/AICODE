# 用户权限管理功能点

> 创建时间：2026-04-25

## 一、功能概述

系统管理模块提供用户、角色、权限的管理能力，支持多级分角色的精细化权限配置。

## 二、用户管理

### 2.1 功能清单

| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| 用户列表 | 查看所有用户 | P0 |
| 用户搜索 | 按用户名/姓名搜索 | P0 |
| 创建用户 | 创建新用户账号 | P0 |
| 编辑用户 | 修改用户信息 | P0 |
| 删除用户 | 禁用/删除用户 | P0 |
| 分配角色 | 为用户分配角色 | P0 |
| 组织筛选 | 按组织筛选用户 | P1 |
| 角色筛选 | 按角色筛选用户 | P1 |

### 2.2 用户信息

| 字段 | 说明 |
|-----|------|
| 用户名 | 唯一标识 |
| 姓名 | 真实姓名 |
| 邮箱 | 联系邮箱 |
| 手机号 | 联系方式 |
| 所属组织 | 所属单位 |
| 角色 | 拥有的角色 |
| 状态 | 启用/禁用 |

### 3.3 权限配置

系统预置权限：

**系统管理模块**
- system:user:list - 用户列表查看
- system:user:read - 用户详情查看
- system:user:create - 用户创建
- system:user:update - 用户更新
- system:user:delete - 用户删除
- system:role:list - 角色列表查看
- system:role:read - 角色详情查看
- system:role:create - 角色创建
- system:role:update - 角色更新
- system:role:delete - 角色删除
- system:permission:list - 权限列表查看

**知识管理模块**
- knowledge:search - 知识搜索
- knowledge:read - 知识查看
- knowledge:create - 知识创建
- knowledge:update - 知识更新
- knowledge:delete - 知识删除
- knowledge:approve - 知识审批

**门户配置模块**
- portal:config - 门户配置
- portal:view - 门户查看

## 六、数据权限控制

### 6.1 多维度权限

1. **模块权限**：用户能否使用某功能
2. **数据权限**：用户能访问哪些知识
3. **组织范围**：用户在哪个地域有效

### 6.2 知识多视角实现

- 海口用户 → 只能看海口数据
- 省级用户 → 可看全省+国家法规
- 执法人员 → 可看执法相关分类
