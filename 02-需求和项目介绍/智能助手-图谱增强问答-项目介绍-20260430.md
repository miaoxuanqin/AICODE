## 海南省住建知识库系统

### 项目简介

住建知识库系统是面向海南省住建和综合执法领域的智能化知识管理平台，集成知识图谱、智能问答、语义搜索、大模型辅助等能力，为执法人员提供高效、准确的知识检索、智能问答与决策支持服务。

系统自 2026 年初启动建设，目前已实现知识管理、智能搜索、图谱增强问答、执法辅助等核心功能模块，覆盖知识从入库、解析、存储、检索到智能问答的全生命周期管理。

### 业务背景

随着住建和综合执法领域法规政策日益繁多、业务场景日趋复杂，传统的关键词检索方式已难以满足一线执法人员快速准确定位依据、高效处理违规行为的需求。系统旨在通过智能化技术手段，构建结构化的知识管理体系，降低知识获取门槛，提升业务处理效率。

### 核心功能模块

**1. 知识管理**
- 支持 Word/PDF 文档自动解析入库（自动提取正文内容）
- 富文本编辑器手动创建知识（支持加粗、斜体、列表等格式）
- 知识关联关系可视化展示
- 知识收藏、评论功能
- 完整的 CRUD 操作与操作日志审计

**2. 智能搜索**
- 全文检索：基于 Elasticsearch 8.13 的关键词搜索
- 语义搜索：基于 Qdrant 向量库的自然语言相似性匹配
- 热搜词统计：基于 Redis Sorted Set 的实时热词排行
- 搜索结果高亮展示

**3. 图谱增强问答（Graph-RAG）**
- 基于 Neo4j 知识图谱的多跳推理能力
- 实体关系自动抽取（法规、条款、标准、处罚、案例、主体、行为等7类实体）
- 推理过程可视化展示，生成可追溯的推理链条
- 支持路径探索与节点关联分析
- 图谱上下文增强：Neo4j 数据参与 LLM 回答生成，提升回答准确性与引用准确性

**4. 执法智能助手**
- 针对执法场景优化的问答流程
- 关联法规检索与案例匹配
- 违规行为定性分析与处罚依据推荐

**5. 工程监管助手**
- 工程合规性审查支持
- 规范标准智能检索
- 案例匹配与建议生成

### 技术架构

| 层级 | 技术选型 | 说明 |
|-----|---------|------|
| 后端框架 | FastAPI + SQLAlchemy | 高性能异步 API 框架 |
| 前端框架 | Vue3 + Element Plus + Pinia | 响应式组件化开发 |
| 关系数据库 | MySQL | 知识元数据存储 |
| 缓存 | Redis | 会话管理、热词统计 |
| 搜索引擎 | Elasticsearch 8.13 | 全文检索 |
| 向量数据库 | Qdrant | 语义搜索支持 |
| 图数据库 | Neo4j | 知识图谱存储与多跳推理 |
| 文件存储 | MinIO (S3兼容) | 原始文档存储 |
| 大模型 | MiniMax-M2.7 / Anthropic | 智能问答与实体抽取 |

### 技术亮点

1. **四位一体知识存储**：关系型（MySQL）+ 全文搜索（Elasticsearch）+ 向量搜索（Qdrant）+ 图数据库（Neo4j），兼顾结构化查询与语义理解

2. **Graph-RAG 架构**：将传统 RAG 与知识图谱结合，实现多跳推理与可解释性。Neo4j 图谱数据不仅用于前端可视化，还参与 LLM 回答生成，显著提升回答质量

3. **推理路径可视化**：将大模型思考过程以结构化步骤展示（理解问题 → 知识检索 → 多跳推理 → 生成回答），增强用户信任度

4. **自动实体关系抽取**：知识入库时自动调用 LLM 抽取实体与关系，支撑动态图谱构建，降低人工维护成本

5. **书名号智能匹配**：自动处理法规名称中《》的有无差异，如"安全生产许可证条例"自动匹配"《安全生产许可证条例》"

### 应用场景

- 执法人员查询法规依据与案例参考
- 违规行为定性与处罚标准检索
- 智能问答与知识推荐
- 法规引用关系追溯与图谱可视化
- 工程合规性审查与建议

### 项目成果

系统于 2026 年 4 月完成核心功能开发与数据初始化，已具备以下能力：
- 支持 7 类实体（法规、条款、标准、处罚、案例、主体、行为）的知识图谱构建
- 提供图谱增强的智能问答服务
- 实现搜索、图谱、问答三位一体的知识服务体系
- 支持多角色权限管理（普通用户、管理员）
