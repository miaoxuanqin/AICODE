## 一、功能定位

执法智能助手是**综合执法场景的智能辅助决策系统**，面向一线执法人员。

核心价值：通过 RAG 问答技术，根据执法人员描述的场景，推荐处置方案、引用法律法规、提供类似案例参考。

## 五、技术方案

### 5.1 架构选择

**复用问答助手 RAG 架构**，不做重复建设：

```
执法助手 → qaApi.chat() → qa_service (RAG) → ES搜索 + MiniMax LLM
```

### 5.2 会话管理决策

**独立会话管理**（各助手维护自己会话）

| 方案 | 描述 | 选择 |
|------|------|------|
| 独立会话 | 各助手创建/管理自己的会话 | ✅ 选中 |
| 共享会话 | 执法助手复用车间助手会话 | ❌ 不选 |

**原因**：
1. 职责清晰 - 会话边界明确
2. 用户体验 - 用户在执法助手看到执法相关会话
3. 扩展性 - 可为执法会话加专属字段

### 5.3 会话隔离方案

通过 **category 字段** 区分不同助手的会话：

| category | 用途 | 可见页面 |
|----------|------|----------|
| qa | 问答助手会话 | QAChat.vue |
| law_general | 执法助手会话 | LawAssistant.vue |

**实现**：
- LawAssistant 进入页面时创建 category='law_general' 的会话
- QAChat 加载会话时过滤掉 category='law_general' 的会话

### 5.4 会话管理策略

- **静默管理** - 用户无感知，不显示会话列表
- **自动关联** - 进入执法助手时自动创建 category='law_general' 的会话
- **静默保存消息** - 通过 qaApi.chat() 自动保存

## 七、API 对接

### 7.1 问答接口

```javascript
// 调用方式（与 QAChat.vue 完全一致）
qaApi.chat(question, userId, sessionId)
```

### 7.2 创建会话接口

```javascript
// 执法助手创建专属会话
qaApi.createSession('【执法】新对话', 'law_general')
```

### 7.3 请求参数

| 参数 | 类型 | 说明 |
|------|------|------|
| question | string | 用户问题/场景描述 |
| user_id | string | 用户ID |
| session_id | string | 会话ID（可选，自动创建） |

### 7.4 响应数据

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 问答记录ID |
| answer | string | LLM 生成的回答 |
| cards | array | 知识卡片列表 |
| related_questions | array | 相关问题推荐 |
| session_id | string | 会话ID |
