# 知识浏览-知识图谱-图谱浏览功能-需求

> 创建日期：2026-04-28

---

## 一、功能概述

### 1.1 背景

当前系统已有"图谱增强问答"功能，用户在问答过程中可以看到与当前问题相关的局部知识图谱。但这只是**问答驱动**的图谱展示，图谱范围局限在当前问题相关的实体，无法让用户**整体感知**知识体系的结构和关联。

Neo4j 中已存储了从所有知识文档中抽取的实体关系网络，管理员和用户需要一个**独立的图谱浏览页面**，可以自由探索这个知识图谱。

### 1.2 功能定位

| 定位 | 说明 |
|------|------|
| 独立入口 | 左侧导航菜单新增"知识图谱"入口 |
| 可交互探索 | 点击节点展开邻居、支持多跳路径可视化 |
| 可筛选定位 | 按分类（法规/案例/处罚/标准）筛选，支持搜索定位实体 |
| 补充问答 | 与现有"图谱增强问答"互补，问答侧重推理，图谱浏览侧重探索 |

### 1.3 用户价值

- **结构化认知**：帮助用户建立住建领域知识的整体框架
- **关系追溯**：快速找到法规→条款→标准→处罚→案例的关联链条
- **知识发现**：通过图谱探索发现之前不知道的关联知识

## 二、功能需求

### 2.1 图谱浏览页面

入口：**知识浏览 → 知识图谱**

#### 页面布局

```
┌────────────────────────────────────────────────────────────┐
│  知识图谱                                          [全屏] │
├────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────┐  ┌─────────────────┐ │
│  │                                 │  │   图例          │ │
│  │        图谱可视化区域             │  │   ● 法规        │ │
│  │                                 │  │   ● 案例        │ │
│  │    [节点A]──关系──[节点B]        │  │   ● 处罚        │ │
│  │        │                        │  │   ● 标准        │ │
│  │      关系                        │  │   ● 条款        │ │
│  │        ↓                        │  └─────────────────┘ │
│  │    [节点C]                       │  ┌─────────────────┐ │
│  │                                 │  │   统计信息      │ │
│  │                                 │  │   节点: 1,234   │ │
│  │                                 │  │   关系: 3,456   │ │
│  └─────────────────────────────────┘  └─────────────────┘ │
│  ┌────────────────────────────────────────────────────────┐ │
│  │ [搜索框] [法规 ▼] [案例 ▼] [处罚 ▼] [清空筛选]         │ │
│  └────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

#### 核心功能

1. **图谱展示**
   - 使用 D3.js / vis.js / cytoscape.js 渲染力导向图
   - 节点按类型显示不同颜色和图标
   - 边显示关系名称，鼠标悬停高亮

2. **节点交互**
   - 点击节点：显示节点详情面板
   - 双击节点：展开邻居节点（深度1）
   - 拖拽节点：调整布局
   - 缩放平移：支持鼠标滚轮缩放和拖拽平移

3. **节点详情面板**
   - 显示节点名称、类型、描述
   - 列出所有关联关系
   - 点击关联边可跳转目标节点
   - "查看知识详情"跳转链接

4. **筛选功能**
   - 按实体类型（法规/案例/处罚/条款/标准/主体/行为）筛选
   - 按关键词搜索节点
   - 支持多选筛选

5. **统计信息**
   - 显示总节点数、总边数
   - 按类型分布（饼图或柱状图）

### 2.2 路径探索功能

#### 功能入口

在图谱页面工具栏提供"路径探索"按钮，点击弹出模态框。

#### 模态框布局

```
┌────────────────────────────────────────────────────────────┐
│  路径探索                                              [×] │
├────────────────────────────────────────────────────────────┤
│  起始节点：[输入框/选择框                          ]      │
│  目标节点：[输入框/选择框                          ]      │
│  最大深度：[3 ▼]                                            │
│                                                            │
│  [查找路径]                                                │
│                                                            │
│  ┌────────────────────────────────────────────────────────┐│
│  │  路径1：法规A → 包含 → 条款B → 引用 → 标准C              ││
│  │  路径2：法规A → 包含 → 条款B → 规定 → 行为D → 触发 → 处罚E││
│  └────────────────────────────────────────────────────────┘│
└────────────────────────────────────────────────────────────┘
```

#### 核心功能

1. **节点选择**
   - 支持搜索下拉选择已有实体
   - 自动补全

2. **路径查找**
   - 使用 Neo4j BFS/DFS 查找最短路径
   - 显示所有路径（最多显示5条）

3. **路径展示**
   - 在图谱中高亮显示路径
   - 路径列表可点击跳转到对应节点

## 三、数据来源

### 3.1 Neo4j 数据

图谱数据完全来自 Neo4j 数据库，与现有图谱增强问答共用同一套数据。

### 3.2 实体类型

| 类型 | 说明 | 颜色 |
|------|------|------|
| Law | 法规 | #409eff (蓝) |
| Article | 条款 | #1a3a6b (深蓝) |
| Case | 案例 | #67c23a (绿) |
| Penalty | 处罚 | #f56c6c (红) |
| Standard | 标准 | #909399 (灰) |
| Subject | 主体 | #e6a23c (橙) |
| Behavior | 行为 | #9b59b6 (紫) |

### 3.3 关系类型

| 关系 | 说明 | 显示名称 |
|------|------|----------|
| CONTAINS | 法规包含条款 | 包含 |
| REFERS | 条款引用标准 | 引用 |
| DEFINES | 条款规定行为 | 规定 |
| TRIGGERS | 行为触发处罚 | 触发 |
| APPLIES | 案例适用条款 | 适用 |
| INVOLVES | 案例涉及主体 | 涉及 |
| IMPOSES | 处罚施加主体 | 施加 |

## 四、页面交互设计

### 4.1 默认展示

- 页面加载时显示图谱概览（采样展示50个核心节点）
- 不加载全量数据，避免性能问题

### 4.2 交互流程

```
1. 用户打开图谱页面
   ↓
2. 系统查询 Neo4j 统计信息，显示节点/边总数
   ↓
3. 系统采样展示50个核心节点（度数最高的节点）
   ↓
4. 用户可以通过：
   - 筛选类型减少显示范围
   - 搜索定位特定节点
   - 点击节点展开邻居
   ↓
5. 节点展开后，图谱动态扩展
```

### 4.3 性能考虑

| 场景 | 处理方式 |
|------|----------|
| 初始加载 | 采样50个核心节点 |
| 节点展开 | 异步加载邻居（最多加载2层） |
| 搜索定位 | 返回节点并高亮，附近节点 |
| 路径探索 | Neo4j 原生查询，最大深度4 |

## 五、非功能性需求

### 5.1 性能

- 页面初始加载 < 3秒
- 节点点击响应 < 500ms
- 路径查找（深度≤4）< 2秒

### 5.2 兼容性

- 浏览器：Chrome、Firefox、Safari、Edge（最新版）
- 响应式：在 1280×720 分辨率以上可用

### 5.3 可访问性

- 键盘可导航
- 节点详情可朗读

## 六、验收标准

| 验收点 | 标准 |
|--------|------|
| 图谱渲染 | 页面加载后正确渲染节点和边 |
| 节点交互 | 点击节点显示详情，展开邻居 |
| 类型筛选 | 选择类型后正确过滤显示 |
| 搜索定位 | 输入关键词后高亮匹配节点 |
| 路径探索 | 输入起止节点后正确显示路径 |
| 菜单入口 | 左侧导航显示"知识图谱"菜单 |
