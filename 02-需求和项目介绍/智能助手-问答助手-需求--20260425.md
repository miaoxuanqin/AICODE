# 问答助手需求理解

> 创建时间：2026-04-25

## 一、功能定位

问答助手是知识库的智能交互入口，通过 RAG（检索增强生成）架构，让用户以自然语言方式查询知识库中的内容。

## 二、核心价值

| 场景 | 传统方式 | 问答助手 |
|------|----------|----------|
| 查政策 | 搜索 → 筛选 → 阅读文档 | 直接提问 → 获取答案 |
| 找流程 | 搜索 → 找相关文档 → 提炼 | 直接提问 → 获取指引 |
| 问条件 | 搜索 → 定位条款 → 理解 | 直接提问 → 智能解读 |

## 三、技术方案

### 3.1 架构选择

**RAG（Retrieval-Augmented Generation）架构**

```
用户提问
    ↓
[1] 检索层：ES 关键词搜索 → 找到相关知识片段
    ↓
[2] 生成层：LLM + 检索结果 → 生成回答
    ↓
[3] 后处理：格式化输出 + 知识卡片 + 推荐问题
    ↓
返回前端展示
```

### 3.2 关键技术验证结果

| 组件 | 验证结果 | 说明 |
|------|----------|------|
| ES 关键词搜索 | ✅ 可用 | 搜索到 2 条测试数据 |
| MiniMax LLM | ✅ 可用 | 返回格式为 ThinkingBlock + TextBlock |
| 完整流程 | ✅ 跑通 | 从搜索到 LLM 生成全流程验证成功 |

### 3.3 为什么不用向量搜索

当前 MiniMax Embedding API 不可用，故采用纯关键词搜索 + LLM 理解的方案。

LLM 本身具备语义理解能力，关键词检索结果配合 LLM 分析，效果仍然良好。

后续如需升级，可快速接入 Jina AI / OpenAI 等 Embedding 服务。

### 4.1 核心功能

|------|------|--------|

## 六、数据统计

| 指标 | 说明 |
|------|------|
| 累计问答数 | 系统总问答次数 |
| 今日问答数 | 当天问答次数 |
| 满意度 | 点赞/(点赞+点踩) 比例 |
| 热门问题 | 按问答次数排序 |

## 七、与现有模块的关系

```
知识管理 → 知识存储（MySQL + ES）
    ↓
搜索门户 → 关键词搜索（ES）
    ↓
问答助手 → 检索 + LLM 生成（RAG）
```

问答助手依赖搜索服务的检索能力，复用 ES 索引。
