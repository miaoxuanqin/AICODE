## 一、需求背景

根据初设要求，知识管理需要支持多层级分类体系。原来 `category` 是固定4个一级分类的字符串枚举值（law/tech/case/policy），无法满足多层级分类和分类管理的需求。

本次实现：在知识管理页面左侧添加分类树形结构，并提供分类的增删改管理功能。

## 三、后端实现

### 3.1 Category 模型

**文件**: `code/backend/app/models/category.py`

```python
class Category(Base):
    __tablename__ = "categories"

    id = Column(Integer, primary_key=True, autoincrement=True)
    name = Column(String(100), nullable=False)
    parent_id = Column(Integer, ForeignKey("categories.id"), nullable=True, index=True)
    level = Column(Integer, default=1)
    description = Column(Text, nullable=True)
    sort_order = Column(Integer, default=0)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

    children = relationship("Category", backref="parent", remote_side=[id], foreign_keys=[parent_id])
```

### 3.3 Category API

**文件**: `code/backend/app/api/v1/category.py`

| 接口 | 方法 | 说明 |
|------|------|------|
| `/categories` | GET | 获取分类树形列表 |
| `/categories/{id}` | GET | 获取单个分类 |
| `/categories` | POST | 创建分类 |
| `/categories/{id}` | PUT | 更新分类 |
| `/categories/{id}` | DELETE | 删除分类（仅管理员，需无子分类和知识关联） |

**树形结构构建**:
```python
def build_tree(categories, parent_id=None):
    tree = []
    for cat in categories:
        if (parent_id is None and cat.parent_id is None) or \
           (cat.parent_id is not None and str(cat.parent_id) == str(parent_id)):
            node = {
                "id": cat.id,
                "name": cat.name,
                "parent_id": cat.parent_id,
                "level": cat.level,
                "sort_order": cat.sort_order,
                "children": build_tree(categories, cat.id)
            }
            tree.append(node)
    tree.sort(key=lambda x: x.get("sort_order", 0))
    return tree
```

## 五、本次修复的问题

### 5.1 图谱重建异步任务修复

**问题**: rebuild 接口使用 `asyncio.create_task()` 在同步函数中调度异步任务，但同步函数没有活跃的事件循环，导致任务从未执行。

**修复**: 改用 `threading.Thread` 直接启动后台线程：
```python
import threading
thread = threading.Thread(
    target=_sync_knowledge_to_neo4j,
    args=(knowledge_id, content),
    daemon=True
)
thread.start()
```

### 5.2 搜索筛选参数修复

**问题**: 前端 `loadData()` 只传递了 `keyword` 和 `category`，其他筛选参数（type、es_indexed、vector_indexed、graph_indexed）未传递。

**修复**:
1. 后端 list 接口新增 type、es_indexed、vector_indexed、graph_indexed 参数
2. 前端 loadData() 补充传递所有筛选参数

### 5.3 Word文档类型值修正

**问题**: 前端下拉框值为 `doc`，但数据库实际存储为 `docx`。

**修复**: 前端 el-option value 从 `doc` 改为 `docx`。

### 5.4 图谱重建用户体验优化

**问题**: 图谱重建是异步任务，用户点击后无法得知处理结果。

**修复**: 点击重建后，前端每2秒轮询一次状态（最多6次），完成后显示成功/失败提示。
