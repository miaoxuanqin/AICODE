## 二、问题修复记录

### 2.1 NEO4J_AVAILABLE 标志缺失

**问题描述**：
`app/api/v1/graph.py` 尝试导入 `NEO4J_AVAILABLE` 失败

**错误信息**：
```
ImportError: cannot import name 'NEO4J_AVAILABLE' from 'app.services.neo4j_service'
```

**修复方案**：
在 `neo4j_service.py` 开头添加：
```python
NEO4J_AVAILABLE = True
```

### 2.2 Neo4j 5.x 语法不兼容

**问题描述**：
Neo4j 5.x 不支持 `size((n)--()))` 语法

**错误信息**：
```
neo4j.exceptions.CypherSyntaxError: A pattern expression should only be used in order to test the existence of a pattern. It can no longer be used inside the function size()
```

**修复方案**：
```python
# 旧语法
WHERE size((n)--())) > 0
WITH n, size((n)--()) AS degree

# 新语法
WHERE size([(n)--() | 1]) > 0
WITH n, size([(n)--() | 1]) AS degree
```

### 2.3 get_center_nodes 边查询返回空

**问题描述**：
边查询要求两边节点都在 node_names 列表中，导致返回空

**修复方案**：
```python
# 旧查询
WHERE a.name IN $node_names AND b.name IN $node_names

# 新查询（只要求源节点在列表中）
WHERE a.name IN $node_names
```

### 2.4 DataSet 去重错误

**问题描述**：
同一节点 ID 出现在多条边中，导致 `DataSet add item with id already exists` 错误

**修复方案**：
渲染前对节点和边进行去重：
```javascript
const uniqueNodes = Array.from(
  new Map(nodes.value.map(n => [n.id, n])).values()
)
const uniqueEdges = Array.from(
  new Map(edges.value.map(e => [e.id, e])).values()
)
```

### 2.5 字体显示问题

**问题描述**：
深色节点上白色字体看不清

**修复方案**：
```javascript
font: { color: '#ffffff', size: 12, strokeWidth: 2, strokeColor: '#000000' }
```

### 2.6 模板语法错误

**问题描述**：
`Element is missing end tag` at div.graph-container

**原因**：
el-popover 的闭合标签位置不正确，导致 div.graph-container 没有正确闭合

**修复方案**：
重构图例部分，使用 legend-item-wrapper 包装每个 popover：
```html
<div class="graph-legend">
  <div v-for="type in nodeTypes" class="legend-item-wrapper">
    <el-popover>
      <template #reference>
        <div class="legend-item">...</div>
      </template>
      <div class="legend-popover">...</div>
    </el-popover>
  </div>
</div>
```
