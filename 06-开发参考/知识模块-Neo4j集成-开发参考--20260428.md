## 一、环境准备

### 1.1 安装依赖

```bash
cd code/backend
pip install neo4j==5.14.0
```

### 1.2 启动 Neo4j

```bash
# Docker 启动
docker run -d --name neo4j \
  -p 7474:7474 \
  -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/neo4j123 \
  neo4j:5-community

# 验证连接
docker exec neo4j cypher-shell -u neo4j -p neo4j123 "MATCH (n) RETURN count(n)"
```

### 1.3 配置

在 `.env` 文件中添加：

```env
NEO4J_URI=bolt://172.20.36.91:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=neo4j123
```

## 二、服务说明

### 2.1 Neo4jService

**文件**：`code/backend/app/services/neo4j_service.py`

#### 初始化

```python
from app.services.neo4j_service import get_neo4j_service

neo4j = get_neo4j_service()

# 验证连接
if neo4j.verify_connectivity():
    print("Neo4j 连接正常")
```

#### 节点操作

```python
# 插入或更新节点
neo4j.upsert_node(
    label="Law",
    name="噪声污染防治法",
    properties={"code": "GB12345", "category": "law"},
    knowledge_id="knowledge-uuid-123"  # 关联知识
)

# 获取节点
node = neo4j.get_node("Law", "噪声污染防治法")

# 检查节点是否存在
exists = neo4j.node_exists("Law", "噪声污染防治法")

# 删除节点
neo4j.delete_node("Law", "噪声污染防治法")
```

#### 关系操作

```python
# 创建关系
neo4j.create_relation(
    from_name="噪声污染防治法",
    from_label="Law",
    to_name="第2条",
    to_label="Article",
    rel_type="CONTAINS"
)

# 检查关系是否存在
exists = neo4j.relation_exists("噪声防治法", "第2条", "CONTAINS")

# 删除关系
neo4j.delete_relation("噪声防治法", "第2条", "CONTAINS")
```

#### 图谱查询

```python
# 获取实体关联的所有节点和关系
related = neo4j.get_related_entities("噪声污染防治法", depth=2)

# 查找两实体间的所有路径
paths = neo4j.find_paths("噪声污染防治法", "罚款", max_depth=4)

# 获取实体周围的子图
subgraph = neo4j.get_subgraph("噪声污染防治法", depth=2)
# 返回: {"nodes": [...], "edges": [...]}

# 按类型获取实体列表
entities = neo4j.get_entity_by_type("Law", limit=100)
```

#### 统计

```python
# 获取统计信息
stats = neo4j.get_stats()
# 返回: {"node:Law": 10, "node:Article": 25, "rel:CONTAINS": 20, ...}
```

#### 引用清理

```python
# 获取节点被多少知识引用
ref_count = neo4j.get_node_ref_count("Law", "噪声污染防治法")

# 移除知识引用，清理无引用实体
stats = neo4j.remove_knowledge_reference("knowledge-uuid-123")
# 返回: {
#   "entities_checked": 5,
#   "entities_deleted": 2,
#   "entities_retained": 3
# }

# 清空所有数据（危险！）
neo4j.clear_all()
```

## 三、API 接口

### 3.1 Graph API

#### 图谱问答

```bash
# 不使用 Neo4j（默认）
POST /api/v1/graph/qa
{"question": "噪声超标怎么处罚？"}

# 使用 Neo4j 增强推理
POST /api/v1/graph/qa
{"question": "噪声超标怎么处罚？", "use_neo4j": true}
```

#### Neo4j 状态

```bash
GET /api/v1/graph/neo4j/status
# 返回: {"available": true, "stats": {...}, "uri": "bolt://..."}
```

#### 获取实体关联图谱

```bash
GET /api/v1/graph/neo4j/entity/{entity_name}?depth=2
```

#### 查找路径

```bash
GET /api/v1/graph/neo4j/path?from_name=噪声法&to_name=罚款&max_depth=4
```

#### 同步知识到 Neo4j

```bash
POST /api/v1/graph/neo4j/sync/{knowledge_id}?text=法规内容...
```

#### 按类型获取实体

```bash
GET /api/v1/graph/neo4j/entities/{label}?limit=100
```

## 五、实体关系抽取

### 5.1 抽取策略

**两层抽取架构**：

1. **规则快速抽取**（速度快）
   - 正则匹配条款编号：`第[一二三四五六七八九十百千万\d]+[条款项]`
   - 正则匹配法规名称：`《[^》]+》`
   - 正则匹配处罚：`罚款[0-9,]+元`

2. **LLM 语义抽取**（精度高）
   - 理解实体类型
   - 识别隐含关系

### 5.2 实体类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Law | 法规 | 《噪声污染防治法》 |
| Article | 条款 | 第2条、第3条第1款 |
| Standard | 标准 | GB 3096-2008 |
| Penalty | 处罚 | 罚款5000-50000元 |
| Case | 案例 | XX项目噪声超标案 |
| Subject | 主体 | 建设单位、施工单位 |
| Behavior | 行为 | 噪声超标排放 |

### 5.3 关系类型

| 类型 | 说明 | 示例 |
|------|------|------|
| CONTAINS | 法规包含条款 | 噪声法 → 第2条 |
| REFERS | 条款引用标准 | 第2条 → GB3096 |
| DEFINES | 条款规定行为 | 第2条 → 噪声超标 |
| TRIGGERS | 行为触发处罚 | 噪声超标 → 罚款 |
| APPLIES | 案例适用条款 | XX案 → 第2条 |
| INVOLVES | 案例涉及主体 | XX案 → 施工单位 |

### 6.3 查看知识关联

在 Neo4j Browser 中执行：

```cypher
# 查看某知识关联的所有实体
MATCH (n)-[:EXTRACTED_FROM]->(k {knowledge_id: 'xxx'})
RETURN n, labels(n)[0] AS type

# 查看某实体的引用计数
MATCH (n {name: '某法规'})-[:EXTRACTED_FROM]->(k)
RETURN count(k) AS ref_count

# 查看无引用的实体
MATCH (n)
WHERE NOT (n)-[:EXTRACTED_FROM]->()
RETURN n, labels(n)[0] AS type
```

## 七、常见问题

### Q1: Neo4j 连接失败？

检查：
1. Neo4j 是否启动：`docker ps | grep neo4j`
2. 端口是否通：`telnet 172.20.36.91 7687`
3. 认证是否正确：NEO4J_AUTH=neo4j/neo4j123

### Q2: 抽取的实体数量为 0？

检查：
1. 文本内容是否为空
2. LLM API 是否正常
3. 查看日志中的错误信息

### Q3: 删除知识后实体没被清理？

检查：
1. EXTRACTED_FROM 关系是否正确建立
2. ref_count 是否正确计算
3. 查看 remove_knowledge_reference 返回的 stats

### Q4: 关系没有建立？

检查：
1. 实体名称是否完全匹配（区分大小写）
2. from_label 和 to_label 是否正确
3. 查看 create_relation 的返回结果
