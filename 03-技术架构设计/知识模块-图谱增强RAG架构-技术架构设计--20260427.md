> 创建日期：2026-04-27
> 更新日期：2026-04-27

## 1. 概述

图谱增强 RAG（Graph-RAG）在传统 RAG 基础上引入知识图谱，通过实体和关系建立知识之间的关联，在问答时展示多跳推理路径，增强回答的可解释性和完整性。

**与向量 RAG 的区别**：

| 维度 | 向量 RAG | 图谱增强 RAG |
|------|----------|--------------|
| 检索方式 | 语义相似度 | 图遍历 + 语义 |
| 推理能力 | 单跳 | 多跳 |
| 结果展示 | 知识卡片列表 | 推理路径图 |
| 可解释性 | 低 | 高 |

| 类型 | Label | 说明 | 示例 |
|------|-------|------|------|
| 问题 | Question | 问题场景 | 脚手架坠落事故 |
| 法规 | Law | 法律法规 | 建设工程安全生产管理条例 |

| 案例 | Case | 执法案例 | 2023年XX工地事故 |

| 标准 | Standard | 技术标准 | 建筑施工安全检查标准 |

**关系类型**：

| 关系 | Type | 说明 | 示例 |
|------|------|------|------|
| 涉及 | INVOLVES | 问题涉及的法规 | (问题)-[涉及]->(法规) |
| 包含 | CONTAINS | 法规包含条款 | (法规)-[包含]->(条款) |
| 导致 | LEADS_TO | 条款导致处罚 | (条款)-[导致]->(处罚) |
| 类似案例 | SIMILAR_CASE | 类似案例关联 | (问题)-[类似案例]->(案例) |
| 处罚依据 | BASIS | 案例的处罚依据 | (案例)-[处罚依据]->(法规) |
| 支撑 | SUPPORTS | 标准支撑条款 | (标准)-[支撑]->(条款) |
| 程序依据 | PROCEDURE_BASIS | 处罚的程序依据 | (处罚)-[程序依据]->(法律) |

### 3.3 Cypher 查询示例

```cypher
// 创建实体
CREATE (q:Question {id: 'q1', name: '脚手架坠落事故', description: '工地脚手架搭建不符合规范导致坠落事故'})
CREATE (l:Law {id: 'n1', name: '建设工程安全生产管理条例', description: '国务院发布的安全生产管理条例'})
CREATE (a:Article {id: 'n2', name: '第78条', description: '未按规定设置安全防护设施的处罚规定'})
CREATE (c:Case {id: 'n3', name: '2023年XX工地事故', description: '某工地脚手架坠落致2人死亡事故'})
CREATE (p:Penalty {id: 'n6', name: '罚款15万元', description: '依据安全生产管理条例第78条的处罚决定'})

// 创建关系
CREATE (q)-[:INVOLVES]->(l)
CREATE (l)-[:CONTAINS]->(a)
CREATE (a)-[:LEADS_TO]->(p)
CREATE (q)-[:SIMILAR_CASE]->(c)
CREATE (c)-[:BASIS]->(l)
```

**多跳查询（3跳）**：

```cypher
// 查询从某个问题出发的多跳关系
MATCH path = (start {name: '脚手架坠落事故'})-[*1..3]->(end)
RETURN path, nodes(path) as entities, relationships(path) as relations
```

**查询某个节点的所有关联**：

```cypher
MATCH (n {name: '第78条'})-[r]-(connected)
RETURN n, r, connected
```

**查询推理路径**：

```cypher
// 查询从问题到处罚的推理路径
MATCH path = (q:Question)-[:INVOLVES|SIMILAR_CASE*1..2]->(l:Law)-[:CONTAINS]->(a:Article)-[:LEADS_TO]->(p:Penalty)
WHERE q.name = '脚手架坠落事故'
RETURN path
```

### 3.4 实体抽取

| 方案 | 选型 | 说明 |
|------|------|------|
| Phase1 | LLM API | 复用现有 MiniMax M2.7 |
| Phase2 | 专用抽取模型 | 训练垂类实体抽取模型 |

**LLM 实体抽取 Prompt**：
```
你是一个实体抽取助手。从以下问题中提取实体，并以JSON格式返回。

问题：{user_question}

要求：
1. 识别实体类型：question(问题场景)、law(法规)、article(条款)、case(案例)、policy(政策)、penalty(处罚)、standard(标准)
2. 返回格式：{"entities": [{"name": "实体名", "type": "类型"}, ...]}

只返回JSON，不要其他内容。
```

### 3.5 图遍历算法（Neo4j 实现）

```python
def multi_hop_traverse(start_name: str, max_hops: int = 3):
    """使用 Cypher 进行多跳遍历"""
    query = f"""
    MATCH path = (start {{name: $start_name}})-[r*1..{max_hops}]->(end)
    WITH path, nodes(path) as ns, rels(path) as rs
    UNWIND ns as node
    UNWIND rs as rel
    RETURN DISTINCT node, rel
    """
    result = neo4j.query(query, {"start_name": start_name})
    return result
```

## 4. 接口设计

### 4.1 图谱查询接口

```
GET /api/v1/graph/search
```

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | string | 是 | 用户问题 |
| max_hops | int | 否 | 最大跳数，默认3 |
| user_id | string | 是 | 用户ID |
