# 知识模块-向量搜索架构-技术架构设计

> 更新日期：2026-04-26

## 1. 概述

向量搜索架构为知识库提供语义搜索能力，基于 RAG (Retrieval-Augmented Generation) 模式实现。用户问题通过 Embedding 模型转换为向量，在 Qdrant 向量数据库中检索相似知识，再结合 LLM 生成回答。

## 4. 搜索类型

| 类型 | 搜索方式 | 适用场景 |
|-----|---------|---------|
| `keyword` | ES multi_match 关键词搜索 | 精确匹配术语 |
| `vector` | Embedding → Qdrant → ES | 语义理解搜索 |
| `hybrid` | 两者合并，去重 | 综合搜索 |

## 5. 关键代码

### 5.1 向量索引 (search_service.py)

```python
def index_vector(self, knowledge_id, title, content, category, user_id):
    """将知识向量存储到 Qdrant"""
    text = f"{title} {content[:1000]}"
    vector = embedding_service.encode_single(text)

    point = PointStruct(
        id=knowledge_id,
        vector=vector,
        payload={
            "title": title,
            "content": content[:500],
            "category": category,
            "user_id": user_id  # 字符串类型
        }
    )
    self.qdrant.upsert(collection_name="knowledge", points=[point])
```

### 5.2 向量搜索 (search_service.py)

```python
def search_vector(self, query_vector, user_id, category=None, limit=5):
    """向量搜索（Qdrant），返回知识ID列表"""
    from qdrant_client.models import Filter, FieldCondition, MatchValue

    must_conditions = [
        FieldCondition(key="user_id", match=MatchValue(value=user_id))
    ]
    if category:
        must_conditions.append(FieldCondition(key="category", match=MatchValue(value=category)))

    search_filter = Filter(must=must_conditions)

    results = self.qdrant.query_points(
        collection_name="knowledge",
        query=query_vector,
        query_filter=search_filter,
        limit=limit
    )

    return [hit.id for hit in results.points]
```
