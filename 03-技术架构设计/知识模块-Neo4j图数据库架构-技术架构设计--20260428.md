## 1. 概述

本文档描述 Neo4j 图数据库在知识模块中的架构设计，用于支撑图谱增强问答（Graph-RAG）功能。

### 1.1 设计目标

1. **持久化存储**：实体关系数据持久化，不依赖内存
2. **高效查询**：支持复杂多跳图遍历查询
3. **知识复用**：实体关系可被多个功能复用
4. **可解释推理**：通过图路径展示推理过程
5. **智能清理**：删除知识时自动清理无引用实体

## 3. 数据模型

### 3.1 节点设计

| 节点类型 | Label | 唯一标识 | 主要属性 |
|---------|-------|---------|---------|
| 法规 | `:Law` | name | code, source, category |
| 条款 | `:Article` | name | content, law_name |
| 标准 | `:Standard` | code | name, category |
| 处罚 | `:Penalty` | name | type, amount, unit |
| 案例 | `:Case` | name | case_no, result, case_date |
| 主体 | `:Subject` | name | type, credit_level |
| 行为 | `:Behavior` | name | description, severity |

### 3.2 关系设计

| 关系 | Type | 起点 | 终点 | 属性 |
|------|------|------|------|------|
| 包含 | `:CONTAINS` | Law | Article | article_no |
| 引用 | `:REFERS` | Article | Standard | reference_type |
| 规定 | `:DEFINES` | Article | Behavior | - |
| 触发 | `:TRIGGERS` | Behavior | Penalty | conditions |
| 适用 | `:APPLIES` | Case | Article | case_no |
| 涉及 | `:INVOLVES` | Case | Subject | role |
| 施加 | `:IMPOSES` | Penalty | Subject | amount |
| **抽取自** | **:EXTRACTED_FROM** | **Entity** | **Knowledge** | **knowledge_id** |

### 3.3 EXTRACTED_FROM 关系

这是核心的引用追踪关系：

```
实体A -[EXTRACTED_FROM]-> 知识1
实体A -[EXTRACTED_FROM]-> 知识2  (同一实体被多个知识引用)
```

**用途**：
- 追踪实体来源
- 删除知识时判断是否需要删除实体

## 5. 知识同步流程

### 5.1 新增知识

```
知识上传/创建
    │
    ▼
┌─────────────────────────────────────┐
│  1. 存入 MySQL/ES/Qdrant           │
│  2. 立即返回（不阻塞）              │
└────────────┬────────────────────────┘
             │ 异步
             ▼
┌─────────────────────────────────────┐
│  graph_extractor.sync_to_neo4j()   │
│                                     │
│  1. 规则抽取实体                    │
│  2. LLM 抽取实体和关系              │
│  3. 合并去重                        │
│  4. 写入 Neo4j                     │
│     - MERGE 节点                   │
│     - 创建 EXTRACTED_FROM 关系      │
└─────────────────────────────────────┘
```

### 5.2 删除知识

```
知识删除
    │
    ▼
┌─────────────────────────────────────┐
│  1. 删除 MinIO 文件                 │
│  2. 删除 ES/Qdrant 索引            │
│  3. 删除 MySQL 记录                │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│  graph_extractor.delete_from_neo4j()│
│                                     │
│  1. 查找关联实体                    │
│     (通过 EXTRACTED_FROM 关系)     │
│  2. 对每个实体：                    │
│     - 移除 EXTRACTED_FROM 关系     │
│     - 检查 ref_count                │
│       - == 0: 删除实体及关系        │
│       - > 0: 保留实体              │
└─────────────────────────────────────┘
```

## 6. 删除逻辑详解

### 6.1 remove_knowledge_reference 流程

```python
def remove_knowledge_reference(self, knowledge_id: str) -> Dict[str, Any]:
    stats = {"entities_checked": 0, "entities_deleted": 0, ...}

    # 1. 找到该知识引用的所有实体
    query = """
    MATCH (n)-[r:EXTRACTED_FROM]->(k {knowledge_id: $knowledge_id})
    RETURN n, labels(n)[0] AS label
    """

    # 2. 对每个实体
    for record in result:
        # 删除 EXTRACTED_FROM 关系
        DELETE r

        # 检查是否还有其他引用
        ref_count = MATCH (n)-[:EXTRACTED_FROM]->() RETURN count(*)

        if ref_count == 0:
            # 无引用了，删除实体
            DETACH DELETE n
            stats["entities_deleted"] += 1
        else:
            # 还有其他引用，保留
            stats["entities_retained"] += 1

    return stats
```

### 6.2 示例

```
场景：实体"噪声污染防治法"被 知识A 和 知识B 引用

删除知识A时：
1. 找到"噪声污染防治法"实体
2. 移除 A-[:EXTRACTED_FROM]->实体 关系
3. ref_count = 1 (还有 B 引用)
4. 实体保留

删除知识B时：
1. 找到"噪声污染防治法"实体
2. 移除 B-[:EXTRACTED_FROM]->实体 关系
3. ref_count = 0 (无任何引用)
4. 删除"噪声污染防治法"实体及所有关系
```

## 7. 部署架构

### 7.1 Docker 部署

```yaml
# docker-compose.yml
version: '3.8'

services:
  neo4j:
    image: neo4j:5-community
    ports:
      - "7474:7474"
      - "7687:7687"
    volumes:
      - neo4j_data:/data
    environment:
      - NEO4J_AUTH=neo4j/neo4j123
      - NEO4J_dbms_memory_heap_initial__size=512m
      - NEO4J_dbms_memory_heap_max__size=2g

volumes:
  neo4j_data:
```

### 7.2 启动命令

```bash
# 启动 Neo4j
docker run -d --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/neo4j123 \
  neo4j:5-community

# 验证连接
docker exec neo4j cypher-shell -u neo4j -p neo4j123 "MATCH (n) RETURN count(n)"
```
