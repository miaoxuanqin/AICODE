# 知识模块-向量搜索架构-技术架构设计

> 更新日期：2026-04-27

## 1. 概述

向量搜索架构为知识库提供语义搜索能力，基于 RAG (Retrieval-Augmented Generation) 模式实现。用户问题通过 Embedding 模型转换为向量，在 Qdrant 向量数据库中检索相似知识，再结合 LLM 生成回答。

**2026-04-27 更新：实现文档全文分片向量化，替代之前的单向量方案。**

## 5. 关键代码

### 5.1 分片索引 (search_service.py) - 2026-04-27 更新

```python
def index_vector(self, knowledge_id, title, content, category, user_id):
    """将知识向量存储到 Qdrant（支持分片）"""
    chunk_size = 500   # 每个 chunk 的字符数
    chunk_overlap = 50 # chunk 重叠字符数

    # 文本分片
    chunks = self._split_into_chunks(content, chunk_size, chunk_overlap)

    if not chunks:
        # 内容为空，只用标题生成一个向量
        vector = embedding_service.encode_single(title)
        point = PointStruct(
            id=knowledge_id,
            vector=vector,
            payload={..., "chunk_index": 0, "total_chunks": 1, "knowledge_id": knowledge_id}
        )
        self.qdrant.upsert(collection_name="knowledge", points=[point])
        return

    # 批量生成向量
    texts_with_index = [f"{title} 第{i+1}段: {chunk}" for i, chunk in enumerate(chunks)]
    vectors = embedding_service.embed_batch(texts_with_index)

    # 构建 points（每个 chunk 一个独立 UUID）
    points = []
    for i, (chunk, vector) in enumerate(zip(chunks, vectors)):
        chunk_id = str(uuid.uuid4())  # Qdrant 只接受 UUID
        point = PointStruct(
            id=chunk_id,
            vector=vector,
            payload={
                "title": title,
                "content": chunk,
                "category": category,
                "user_id": str(user_id),  # 统一字符串类型
                "chunk_index": i,
                "total_chunks": len(chunks),
                "knowledge_id": knowledge_id
            }
        )
        points.append(point)

    self.qdrant.upsert(collection_name="knowledge", points=points)

def _split_into_chunks(self, text: str, chunk_size: int, overlap: int) -> List[str]:
    """将文本分割成重叠的 chunks"""
    chunks = []
    start = 0
    text_len = len(text)

    while start < text_len:
        end = start + chunk_size
        chunk = text[start:end]
        # 去除空行，保持阅读连贯性
        chunk = '\n'.join([line.strip() for line in chunk.split('\n') if line.strip()])
        if chunk:
            chunks.append(chunk)
        start += chunk_size - overlap
        if overlap >= chunk_size:
            break

    return chunks
```

### 5.2 向量搜索（自动去重）- 2026-04-27 更新

```python
def search_vector(self, query_vector, user_id, category=None, limit=5):
    """向量搜索（Qdrant），返回去重后的知识ID列表"""
    from qdrant_client.models import Filter, FieldCondition, MatchValue

    must_conditions = [
        FieldCondition(key="user_id", match=MatchValue(value=str(user_id)))  # 字符串匹配
    ]
    if category:
        must_conditions.append(FieldCondition(key="category", match=MatchValue(value=category)))

    search_filter = Filter(must=must_conditions)

    # 扩大搜索范围（一个知识有多个 chunk）
    search_limit = limit * 3

    results = self.qdrant.query_points(
        collection_name="knowledge",
        query=query_vector,
        query_filter=search_filter,
        limit=search_limit
    )

    # 按 knowledge_id 去重
    knowledge_ids = []
    seen = set()
    for hit in results.points:
        kid = hit.payload.get("knowledge_id") or hit.id
        if kid not in seen:
            seen.add(kid)
            knowledge_ids.append(kid)
            if len(knowledge_ids) >= limit:
                break

    return knowledge_ids
```

## 8. 分片策略说明

### 为什么需要分片？

1. **保留更多信息**：原来只取前 1000 字，现在可以处理整个文档
2. **段落级检索**：可以匹配到文档中最相关的具体段落
3. **上下文保持**：50 字符重叠确保段落边界不会切断语义

### 分片后检索流程

```
用户问题 → Embedding → Qdrant 向量搜索 → 多个 chunk 匹配
                                              ↓
                                      按 knowledge_id 去重
                                              ↓
                                      返回独立知识 ID 列表
                                              ↓
                                      ES 获取完整知识详情
```

### 对 LLM Prompt 的影响

分片后，每个知识在 prompt 中仍然只出现一次（因为去重了）。检索时可能匹配到该知识的多个 chunk，但最终只取一个知识构建 prompt。
