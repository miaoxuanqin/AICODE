# AI入口

## 你的角色

你需要基于本目录中的项目资料，协助生成、完善和维护一个用于培训演示的 Demo 项目。

## 工作目标

- 正确理解项目需求和范围
- 基于设计文档实现可运行的 Demo
- 遵守公司开发规范和参考约束
- 在实现后补充必要测试并更新项目进度

## 工作顺序

1. 阅读 `02-需求和项目介绍/README.md`
2. 阅读 `03-技术架构设计/README.md`
3. 阅读 `04-功能详细设计/README.md`
4. 根据任务需要查阅 `06-开发参考/` 中的外部资料
5. 在编码前确认 `07-开发规范/README.md` 和相关规范文件
6. 完成开发后参考 `05-测试用例设计/README.md` 进行验证
7. 将结果同步到 `08-项目进度/README.md`

## 输出要求

- 代码应优先保证可运行、可演示、可理解
- 新增功能应与需求、架构、功能设计保持一致
- 不明确的地方应优先依据已有文档推断，不要随意发散
- 如某项设计缺失，应先补充设计说明，再开始实现
- 所有实际代码、脚本和测试文件都必须放在 `code/` 目录下

## 代码目录要求

- 不允许在仓库根目录自行新建 `src/`、`app/`、`server/`、`web/` 等代码目录
- `code/` 默认作为 Node 服务工程根目录使用
- 应用代码默认放在 `code/` 下组织
- 数据库脚本应放在 `code/sql/`
- 测试代码和测试脚本应放在 `code/test/`
- 如需使用 `src/`，应创建在 `code/src/` 下，不得直接建在仓库根目录

## 重要约束



- 数据库开发需遵循 `07-开发规范/01-数据库开发规范.md`
- TypeScript 开发需遵循 `07-开发规范/02-TypeScript开发规范.md`
- 未经文档明确说明，不要擅自改变现有目录结构约定

## 本地环境

本项目默认在本地完成版本管理和开发验证，不要求提交到远程 Git 平台。初始化本地环境时，可参考 `06-开发参考/01-开发环境配置.md` 和 `06-开发参考/05-AI本地初始化建议.md`。
