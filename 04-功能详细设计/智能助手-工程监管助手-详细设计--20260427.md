# 工程监管助手详细设计

> 最后更新：2026-04-27

## 页面概述
- **菜单路径**：智能助手 > 工程监管助手
- **功能**：基于知识图谱的工程质量安全问题定位与处置推荐
- **会话 category**：`supervise`

## 页面结构
```
┌─────────────────────────────────────────────────────────────┐
│  工程监管智能助手                                              │
│  基于知识图谱的工程质量安全问题定位与处置推荐                        │
├───────────────────────────────────┬─────────────────────────┤
│                                   │  常见问题类型              │
│                                   │  (混凝土/钢筋/防水/脚手架)   │

│     (聊天消息 + 知识卡片)           │  关联知识                 │

│                                   │  最近监管记录               │
├───────────────────────────────────┴─────────────────────────┤
│  输入框 (textarea) + 发送按钮                                 │
└─────────────────────────────────────────────────────────────┘
```

## 功能详情

### 1. 对话功能
- **消息渲染**：支持 Markdown 格式
- **知识卡片**：显示检索到的相关知识
- **Ctrl+Enter**：快捷发送

### 2. 问题类型模板
预定义工程问题类型：
- 混凝土强度不足
- 钢筋配置不规范
- 防水工程缺陷
- 脚手架安全隐患
- 施工缝处理不当
- 材料以次充好

### 3. 关联知识
- 根据最后一条用户消息动态加载
- 显示知识分类标签和标题

### 4. 最近监管记录
- 复用会话历史中的用户问题
- 显示处理状态标签

## API 接口

### 会话管理
| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取会话列表 | GET | /api/v1/qa/sessions | 过滤 category='supervise' |
| 创建会话 | POST | /api/v1/qa/session | category='supervise' |
| 获取会话详情 | GET | /api/v1/qa/session/{id} | 含消息历史 |
| 清空会话消息 | DELETE | /api/v1/qa/session/{id}/messages |

## 样式规范

### 政务蓝主题
- 主色：`#1a3a6b`
- 辅色：`#b8860b`（金色点缀）
- 背景：`#f0f4f8`

### 消息气泡
- 用户消息：政务蓝背景，白色文字，右侧金色边框
- 助手消息：白色背景，左侧蓝色边框

### 知识卡片
- 顶部蓝色边框
- 悬停：金色边框 + 上浮效果

## 2026-04-27 修复内容

### 1. 卡片字段修复
- **问题**：前端期望 `card.type`，后端只返回 `card.category`
- **解决**：后端同时返回 `type` 和 `category` 两个字段

### 2. 会话初始化错误处理
- **问题**：`getSessions` 失败时未创建新会话
- **解决**：catch 块中添加创建新会话逻辑

### 3. 会话加载错误处理
- **问题**：`loadSessionHistory` 失败导致异常
- **解决**：catch 块中设置 `messages.value = []`

## 与执法助手差异

| 功能 | 执法智能助手 | 工程监管助手 |
|------|-------------|-------------|
| category | `law_general` | `supervise` |
| 搜索大小 | 5 | 8 |
| 问题模板 | 执法场景 | 工程质量 |
| 侧边栏 | 场景/知识/案例 | 问题类型/知识/记录 |
| 附件上传 | 支持 | 不支持 |
