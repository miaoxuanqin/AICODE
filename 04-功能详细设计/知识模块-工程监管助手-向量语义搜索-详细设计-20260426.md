# 知识模块-工程监管助手-向量语义搜索详细设计

> 更新日期：2026-04-26

## 1. 功能概述

工程监管助手采用向量语义搜索技术，实现对工程质量安全问题的智能问答。与传统关键词搜索不同，向量搜索能够理解语义关联，即使表述不同但含义相似的问题也能被正确检索。

## 2. 技术方案

### 2.1 向量搜索流程

```
用户问题
    ↓
Embedding 模型转向量 (text2vec-base-chinese)
    ↓
Qdrant 向量相似度搜索
    ↓
返回最相似的知识片段
    ↓
构建 Prompt → LLM 生成回答
```

### 2.2 Embedding 服务

**部署位置：** WSL (Ubuntu 24.04)
**服务地址：** http://172.20.36.91:8001
**模型：** shibing624/text2vec-base-chinese
**向量维度：** 768

**接口：**
```python
POST /embed
Request: ["文本列表"]
Response: {"embeddings": [[float, float, ...]]}

GET /health
Response: {"status": "ok"}
```

### 2.3 混合搜索策略

根据 category 决定搜索方式：

| Category | 搜索方式 | 说明 |
|----------|---------|------|
| supervise | 向量搜索 | 语义理解能力强 |
| qa | ES 关键词 | 精确匹配 |
| law_general | ES 关键词 | 精确匹配 |

## 3. 代码实现

### 3.1 Embedding 服务 (embedding_service.py)

```python
class EmbeddingService:
    # 本地 WSL 服务
    LOCAL_URL = "http://172.20.36.91:8001"

    # MiniMax API (云端备选)
    MINIMAX_URL = "https://api.minimax.chat/v1/embeddings"

    def __init__(self, mode: str = "local"):
        self.mode = mode  # "local" 或 "minimax"

    def encode_single(self, text: str) -> List[float]:
        """单个文本转为向量"""
        if self._use_local():
            return self._embed_local([text])[0]
        return self._embed_minimax([text], "query")[0]
```

### 3.3 问答服务 (qa_service.py)

```python
class QAService:
    CATEGORY_CONFIG = {
        "supervise": {
            "name": "工程监管助手",

            """,
            "search_size": 8,  # 工程监管需要更多参考资料
        }
    }

    def ask(self, question: str, user_id: str, session_id: str, db: Session):
        # 获取 category
        current_category = session.category or "qa"

        # 根据 category 选择搜索方式
        if current_category == "supervise":
            # 向量搜索

```

```
    ↓

    ↓

    ↓

    ↓
Embedding 生成向量
    ↓
Qdrant 存储向量
    ↓
完成索引
```

## 10. 已知限制

1. **首次加载慢**：Embedding 模型首次加载约需 10-30 秒
2. **向量维度固定**：text2vec-base-chinese 固定 768 维
3. **Qdrant 需预先配置**：collection 和向量维度需匹配
4. **WSL 服务独立性**：Embedding 服务独立运行，后端仅通过 HTTP 调用
