## 二、后端 API 设计

### 2.1 复用现有 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/graph/neo4j/status` | GET | 查询 Neo4j 连接状态 |
| `/api/v1/graph/neo4j/entity/{name}` | GET | 获取实体关联图谱 |
| `/api/v1/graph/neo4j/path` | GET | 查找两实体间路径 |
| `/api/v1/graph/neo4j/entities/{label}` | GET | 按类型获取实体 |

### 2.2 新增 API

#### 2.2.1 图谱统计信息

**接口**：`GET /api/v1/graph/explorer/stats`

**响应**：
```json
{
  "available": true,
  "total_nodes": 1234,
  "total_edges": 5678,
  "by_type": {
    "Law": 120,
    "Article": 400,
    "Case": 150,
    "Penalty": 80,
    "Standard": 200,
    "Subject": 184,
    "Behavior": 100
  }
}
```

#### 2.2.2 获取中心节点（采样展示）

**接口**：`GET /api/v1/graph/explorer/center`

**参数**：
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| limit | int | 50 | 返回节点数 |

**响应**：
```json
{
  "nodes": [
    {
      "id": "node_1",
      "name": "《建设工程安全生产管理条例》",
      "label": "Law",
      "degree": 25
    }
  ],
  "edges": [
    {
      "source": "node_1",
      "target": "node_2",
      "type": "CONTAINS"
    }
  ]
}
```

#### 2.2.3 获取节点邻居

**接口**：`GET /api/v1/graph/explorer/neighbors/{node_id}`

**参数**：
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| depth | int | 1 | 展开深度（1-2） |

**响应**：
```json
{
  "nodes": [...],
  "edges": [...]
}
```

#### 2.2.4 搜索实体

**接口**：`GET /api/v1/graph/explorer/search`

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| q | string | 搜索关键词 |
| label | string | 可选，按类型筛选 |

**响应**：
```json
{
  "results": [
    {
      "id": "node_xxx",
      "name": "《建设工程安全生产管理条例》",
      "label": "Law",
      "matched_on": "name"
    }
  ]
}
```

## 四、数据库查询设计

### 4.1 获取中心节点（按度数采样）

```cypher
// 获取度数最高的 N 个节点
MATCH (n)
WHERE n:Law OR n:Article OR n:Case OR n:Penalty
   OR n:Standard OR n:Subject OR n:Behavior
WITH n, size((n)--()) AS degree
ORDER BY degree DESC
LIMIT 50

// 获取这些节点的邻居边
MATCH (n)-[r]-(m)
WHERE n IN fetched_nodes
RETURN n, r, m
```

### 4.2 获取邻居节点

```cypher
// 获取指定节点的邻居（深度1）
MATCH (center:Law {name: $name})-[r]-(neighbor)
RETURN center, r, neighbor

// 深度2
MATCH (center:Law {name: $name})-[r1]-(n1)-[r2]-(n2)
RETURN center, r1, n1, r2, n2
```

### 4.3 路径查找

```cypher
// 查找两节点间所有路径
MATCH path = (a {name: $from})-[:CONTAINS|REFERS|DEFINES|TRIGGERS*1..4]-(b {name: $to})
RETURN path
LIMIT 5
```

## 五、前端组件详细设计

### 5.1 GraphCanvas.vue

**功能**：图谱渲染画布

**props**：
| prop | 类型 | 说明 |
|------|------|------|
| nodes | Array | 节点数据 |
| edges | Array | 边数据 |
| highlightedNodes | Array | 高亮节点 |
| highlightedEdges | Array | 高亮边 |

**emits**：
| event | 说明 |
|-------|------|
| node-click | 节点点击 |
| node-dblclick | 节点双击（展开邻居） |
| edge-click | 边点击 |

**技术实现**：
- 使用 vis-network 的 Network 类
- 力导向布局（physics.enabled: true）
- 双击回调中调用 `/explorer/neighbors` 获取邻居

### 5.2 NodeDetailPanel.vue

**功能**：节点详情侧边栏

**props**：
| prop | 类型 | 说明 |
|------|------|------|
| node | Object | 选中节点 |
| visible | Boolean | 显示状态 |

**显示内容**：
- 节点名称（标题）
- 节点类型（标签）
- 关联关系列表（点击跳转）
- 关联知识列表（点击跳转知识详情）
- 操作按钮（查看详情、在图谱中定位）

### 5.3 PathExplorer.vue

**功能**：路径探索弹窗

**内部状态**：
```javascript
const fromNode = ref('')
const toNode = ref('')
const maxDepth = ref(3)
const paths = ref([])
const loading = ref(false)
```

**搜索流程**：
1. 用户输入起止节点（支持下拉搜索）
2. 点击"查找路径"
3. 调用 `/api/v1/graph/neo4j/path` API
4. 显示路径列表
5. 用户点击路径，在图谱中高亮

### 5.4 GraphFilters.vue

**功能**：筛选工具栏

**组件结构**：
```
┌────────────────────────────────────────────────────────────┐
│ [搜索框: 输入节点名称...]  [类型筛选▼]  [重置]             │
└────────────────────────────────────────────────────────────┘
```

**类型筛选**：
- 多选下拉：Law、Article、Case、Penalty、Standard、Subject、Behavior
- 选择后图谱动态过滤显示

## 七、数据流设计

### 7.1 页面初始化

```
GraphExplorer.vue mounted
    │
    ├── 调用 /graph/neo4j/status 检查连接
    │
    ├── 调用 /graph/explorer/stats 获取统计
    │
    ├── 调用 /graph/explorer/center 获取初始节点
    │
    └── 渲染到 GraphCanvas
```

### 7.2 节点展开

```
用户双击节点
    │
    ├── 调用 /graph/explorer/neighbors/{node_id}
    │
    ├── 合并新节点和边到现有数据
    │
    └── 重新渲染图谱
```

### 7.3 搜索定位

```
用户输入搜索词
    │
    ├── 调用 /graph/explorer/search?q=xxx
    │
    ├── 获取匹配节点
    │
    ├── 高亮匹配节点
    │
    └── 自动居中显示
```

## 九、错误处理

| 场景 | 处理方式 |
|------|----------|
| Neo4j 未连接 | 显示离线状态，提示"图谱服务暂不可用" |
| 节点加载失败 | 显示错误提示，提供重试按钮 |
| 邻居展开失败 | Toast 提示"展开失败"，保留当前图谱 |
| 路径查找超时 | Toast 提示"查找超时，请缩小范围" |
