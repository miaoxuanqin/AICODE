> 创建日期：2026-04-28

## 一、本次优化概述

### 1.1 优化背景

图谱增强问答上线后，用户反馈等待答案时界面缺乏动态效果，推理过程不透明。本次优化重点提升用户体验。

### 1.2 优化内容

| 优化项 | 说明 |
|--------|------|
| Neo4j开关 | 前端添加use_neo4j参数，支持切换Neo4j推理模式 |
| 思考中界面 | 实时展示推理链步骤，替代原有简单进度条 |
| 趣味提示 | 轮换显示趣味提示语，减少等待焦虑 |
| 图谱动态效果 | 节点弹入动画、边绘制动画 |
| 动态图谱生成 | 后端无数据时前端动态生成相关节点 |

## 二、Neo4j开关集成

### 2.1 前端API修改

**文件**：`code/frontend/src/api/index.js`

```javascript
// 修改前
qa: (question) => request.post('/graph/qa', { question })

// 修改后
qa: (question, useNeo4j = false) =>
  request.post('/graph/qa', { question, use_neo4j: useNeo4j })
```

### 2.2 页面开关UI

**文件**：`code/frontend/src/views/graph/GraphQAChat.vue`

```vue

  v-model="useNeo4j"
  active-text="使用Neo4j"
  inactive-text=""

```

### 2.3 调用方式

```javascript
const result = await graphApi.qa(userMsg.content, useNeo4j.value)
```

### 2.4 两种模式差异

| 模式 | 推理方式 | 数据来源 |
|------|---------|---------|
| 关闭 | 本地多跳推理 | Elasticsearch + 本地实体提取 |
| 打开 | Neo4j图数据库推理 | Neo4j图数据库 + 实体关系 |

## 四、图谱动态效果

### 4.1 动画类型

| 动画 | 触发条件 | 效果 |
|------|---------|------|
| `nodeAppear` | 新节点加入 | 缩放弹跳 (0.3 → 1.1 → 1) |
| `edgeDraw` | 新边加入 | 线条绘制 (dashoffset动画) |

### 4.2 CSS动画定义

```css
@keyframes nodeAppear {
  0% {
    opacity: 0;
    transform: scale(0.3);
  }
  50% {
    opacity: 0.7;
    transform: scale(1.1);
  }
  100% {
    opacity: 1;
    transform: scale(1);
  }
}

@keyframes edgeDraw {
  0% {
    stroke-dasharray: 100;
    stroke-dashoffset: 100;
  }
  100% {
    stroke-dasharray: 100;
    stroke-dashoffset: 0;
  }
}
```

### 4.3 SVG transition-group

```vue
<g class="nodes">
  <transition-group name="node-fade">
    <g v-for="node in graphData.nodes" :key="node.id" ...>
      ...
    </g>
  </transition-group>
</g>
```

### 4.4 数据变化监听

```javascript
watch(() => graphData.nodes.length, (newLen) => {
  if (newLen > prevNodeCount && prevNodeCount > 0) {
    nextTick(() => {
      document.querySelectorAll('.node-group').forEach((g, idx) => {
        if (idx >= prevNodeCount) {
          g.classList.add('new-node')
          setTimeout(() => g.classList.remove('new-node'), 600)
        }
      })
    })
  }
  prevNodeCount = newLen
})
```

## 五、动态图谱生成

### 5.1 背景

当前端收到的`graph_data`为空时，根据问题内容动态生成相关图谱。

### 5.2 生成逻辑

```javascript
const generateDynamicGraphData = (question, reasoningChain) => {
  const entities = reasoningChain.flatMap(step => step.entities || [])
  const knowledgeFound = reasoningChain.flatMap(step => step.details?.knowledge_found || [])

  // 问题节点
  nodes.push({ id: 'q1', label: question, type: 'question', x: 200, y: 50, ... })

  // 基于知识生成节点
  uniqueKnowledge.forEach((title, idx) => {
    nodes.push({ id: `n${idx+1}`, label: title, type: types[idx % types.length], ... })
    edges.push({ id: `e${idx+1}`, source: 'q1', target: `n${idx+1}`, label: '涉及' })
  })

  // 基于实体生成额外节点
  uniqueEntities.forEach((entity, idx) => {
    nodes.push({ id: `e${idx+10}`, label: entity, type: 'policy', ... })
  })

  return { nodes, edges }
}
```

### 5.3 调用时机

```javascript
// 发送消息后更新图谱
if (result.graph_data && result.graph_data.nodes?.length > 0) {
  // 使用后端返回的数据
  graphData.nodes = result.graph_data.nodes
} else {
  // 动态生成
  const generated = generateDynamicGraphData(userMsg.content, reasoningChain)
  graphData.nodes = generated.nodes
  graphData.edges = generated.edges
}
```
