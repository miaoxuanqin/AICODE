## 一、概述

### 1.1 功能定位

为问答助手、执法智能助手、工程监管助手、图谱增强问答四个智能助手模块增加"思考过程展示"和"取消请求"功能，提升用户等待体验。

### 1.2 背景

原界面在等待 LLM 生成答案时只显示简单的加载图标和文字，用户体验单调，且无法主动取消正在进行的请求。

## 三、取消功能

### 3.1 功能设计

#### 3.1.1 取消逻辑

| 条件 | 行为 |
|------|------|
| 进度 > 15% 且 < 100% | 显示取消按钮 |
| 点击取消 | 停止动画，移除用户消息，显示"已取消"提示 |
| 请求已完成 | 隐藏取消按钮 |

#### 3.1.2 实现方式

```javascript
// 创建取消控制器
let canceled = false
cancelRequestRef = () => {
  canceled = true
  cancelRequestRef = null
}

// 检查是否取消
if (canceled) return

// 取消请求
const cancelRequest = () => {
  if (cancelRequestRef) {
    cancelRequestRef()
    isThinking.value = false
    stopThinkingAnimation()
    // 移除最后一条用户消息
    const lastMsg = messages.value[messages.value.length - 1]
    if (lastMsg && lastMsg.role === 'user') {
      messages.value.pop()
    }
    ElMessage.info('已取消')
  }
}
```

### 3.2 用户体验

- **可主动放弃**：用户等待时如果觉得问法不对，可以取消重新输入
- **不会卡界面**：取消后立即恢复可输入状态
- **保留上下文**：不会清除会话历史，只是移除当前消息
