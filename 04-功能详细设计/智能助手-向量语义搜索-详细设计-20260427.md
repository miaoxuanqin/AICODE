# 智能助手-向量语义搜索-详细设计

> 更新日期：2026-04-27

## 1. 功能概述

三个智能助手（问答助手、执法智能助手、工程监管助手）统一采用**向量语义搜索**技术，实现对用户问题的智能问答。与传统关键词搜索不同，向量搜索能够理解语义关联，即使表述不同但含义相似的问题也能被正确检索。

**2026-04-27 更新：实现文档全文分片向量化，检索时自动去重。**

## 2. 技术方案

### 2.1 向量搜索流程

```
用户问题
    ↓
Embedding 模型转向量 (text2vec-base-chinese, 768维)
    ↓
Qdrant 向量相似度搜索（多chunk匹配）
    ↓
按 knowledge_id 去重
    ↓
返回最相似的知识片段
    ↓
构建 Prompt → LLM 生成回答
```

### 2.2 文档分片策略（2026-04-27 新增）

**分片参数：**
| 参数 | 值 | 说明 |
|------|-----|------|
| chunk_size | 500 字符 | 每个 chunk 大小 |
| chunk_overlap | 50 字符 | 相邻 chunk 重叠，保持上下文连贯 |
| 向量生成 | 每个 chunk 单独生成 | 标题+段落位置作为前缀 |

**示例：**
```
原文：海南省住建厅行政处罚决定书...（共3000字）

分片结果：
chunk_0: "海南省住建厅行政处罚...第1段: 第一条 为规范..."
chunk_1: "第二条 行政处罚的...第2段: 根据《中华..."
chunk_2: "第三条 当事人应当...第3段: 如有不..."
...
```

### 2.3 统一搜索策略（2026-04-27 更新）

**所有助手统一使用向量搜索，失败时 fallback 到关键词搜索：**


### 2.4 搜索方式对比

| 搜索类型 | 原理 | 优势 | 劣势 |
|----------|------|------|------|
| **向量搜索** | 语义理解，将文本转为向量计算相似度 | 同义词/相关概念能关联 | 需额外 Embedding 服务 |
| **关键词搜索** | BM25 评分，精确匹配关键词 | 精确匹配，速度快 | 字面不符则搜不到 |

### 2.5 演示：向量搜索 vs 关键词搜索

**问题：** `楼板出现裂缝怎么处置？`

| 搜索类型 | 结果 | 说明 |
|----------|------|------|
| **向量搜索** | 违规装修查处案例、钢筋配置不规范 | 语义理解"楼板裂缝"=建筑质量问题 |
| **关键词搜索** | 安全生产相关规定 | 字面匹配"楼板"/"安全" |

**问题：** `混凝土强度不足怎么处理？`

| 搜索类型 | 结果 | 说明 |
|----------|------|------|
| **向量搜索** | 混凝土强度不足的处理措施 ✅ | 精确语义匹配 |
| **关键词搜索** | 混凝土强度不足的处理措施 ✅ | 精确关键词匹配 |

```python
class QAService:

            "search_size": 8,  # 工程监管需要更多参考资料
        }
    }

    def ask(self, question: str, user_id: str, session_id: str, db: Session):
        # 获取对话历史
        history_messages = self.get_session_history(current_session_id, user_id, limit=5, db=db)

        # 获取类目配置
        config = self.CATEGORY_CONFIG.get(current_category, self.CATEGORY_CONFIG["qa"])
        search_size = config["search_size"]

        # 1. 检索相关知识（统一使用向量搜索）
        try:
            query_vector = embedding_service.encode_single(question)
            knowledge_ids = self.search_service.search_vector(
                query_vector=query_vector,
                user_id=user_id,
                limit=search_size
            )
            # 根据 ID 获取完整知识信息...
            search_results = {"items": items, "total": len(items)}
        except Exception as e:
            print(f"向量搜索失败，fallback 到关键词搜索: {e}")
            search_results = self.search_service.search_keyword(
                query=question, user_id=user_id, page=1, page_size=search_size
            )

        # 2. 构建 Prompt
        prompt = self._build_prompt(question, search_results['items'], history_messages, current_category)

        # 3. 调用 LLM
        if search_results['total'] > 0:
            answer = self._call_llm(prompt)

            answer = "抱歉，知识库中没有找到与您问题相关的内容..."
```

### 3.2 Embedding 服务 (embedding_service.py)

```python
class EmbeddingService:
    LOCAL_URL = "http://172.20.36.91:8001"  # WSL Embedding
    MINIMAX_URL = "https://api.minimax.chat/v1/embeddings"

    def encode_single(self, text: str) -> List[float]:
        """单个文本转为向量"""
        if self._use_local():
            return self._embed_local([text])[0]
        return self._embed_minimax([text], "query")[0]

    def embed_batch(self, texts: List[str], emb_type: str = "db") -> List[List[float]]:
        """批量将文本转为向量"""
        if self._use_local():
            return self._embed_local(texts)
        return self._embed_minimax(texts, emb_type)
```

### 3.3 搜索服务 (search_service.py) - 2026-04-27 更新

```python
class SearchService:
    def search_vector(self, query_vector: List[float], user_id: str,
                     category: Optional[str] = None, limit: int = 5) -> List[str]:
        """向量搜索（Qdrant），返回去重后的知识ID列表"""
        results = self.qdrant.query_points(
            collection_name="knowledge",
            query=query_vector,
            query_filter=search_filter,
            limit=limit * 3  # 扩大范围
        )

        # 按 knowledge_id 去重
        knowledge_ids = []
        seen = set()
        for hit in results.points:
            kid = hit.payload.get("knowledge_id") or hit.id
            if kid not in seen:
                seen.add(kid)
                knowledge_ids.append(kid)
                if len(knowledge_ids) >= limit:
                    break

        return knowledge_ids

    def index_vector(self, knowledge_id: str, title: str, content: str,
                     category: str, user_id: str):
        """将知识向量存储到 Qdrant（支持分片）"""
        # 分片
        chunks = self._split_into_chunks(content, chunk_size=500, overlap=50)

        # 批量向量化
        texts_with_index = [f"{title} 第{i+1}段: {chunk}" for i, chunk in enumerate(chunks)]
        vectors = embedding_service.embed_batch(texts_with_index)

        # 存储（每个 chunk 一个 point）
        points = []
        for i, (chunk, vector) in enumerate(zip(chunks, vectors)):
            point = PointStruct(
                id=str(uuid.uuid4()),  # 独立 UUID
                vector=vector,
                payload={
                    "title": title,
                    "content": chunk,
                    "category": category,
                    "user_id": str(user_id),  # 字符串类型
                    "chunk_index": i,
                    "total_chunks": len(chunks),
                    "knowledge_id": knowledge_id
                }
            )
            points.append(point)

        self.qdrant.upsert(collection_name="knowledge", points=points)
```

## 6. 知识索引流程

### 6.1 知识创建/更新时

```
POST /knowledge/upload → index_knowledge (ES) + index_vector (Qdrant分片)
POST /knowledge/manual → index_knowledge (ES) + index_vector (Qdrant分片)
```

**分片索引流程：**
```
content → _split_into_chunks(500, 50) → [chunk1, chunk2, ..., chunkN]
                                                    ↓
                                          embed_batch([title 第1段: chunk1, ...])
                                                    ↓
                                          [vector1, vector2, ..., vectorN]
                                                    ↓
                                          upsert to Qdrant (N points)
```

### 6.2 问答搜索时

```
用户问题 → Embedding → Qdrant向量搜索 → 多chunk匹配
                                              ↓
                                      按 knowledge_id 去重
                                              ↓
                                      返回独立知识ID列表 → ES获取详情 → Prompt → LLM
```

## 8. 已知限制

1. **首次加载慢**：Embedding 模型首次加载约需 10-30 秒
2. **向量维度固定**：text2vec-base-chinese 固定 768 维
3. **Qdrant 需预先配置**：collection 和向量维度需匹配
4. **分片边界**：按固定字符数分割，可能切断句子（但50字符重叠可缓解）
