# 智能助手-图谱增强问答-Neo4j集成方案-详细设计

> 创建日期：2026-04-28

---

## 一、方案概述

### 1.1 背景

当前图谱增强问答（Graph-RAG）使用**内存图谱**存储实体关系，存在以下局限：
- 数据不持久化，重启服务后丢失
- 无法进行复杂的多跳图遍历查询
- 实体关系无法复用，查询效率低

### 1.2 解决方案

引入 **Neo4j 图数据库**，采用**向量检索 + Neo4j 推理**的混合架构：

| 组件 | 作用 | 说明 |
|------|------|------|
| 向量检索 | 快速召回 | 从知识库中找到可能相关的文档 |
| Neo4j | 关系推理 | 理清知识之间的关联关系链 |

### 1.3 为什么适合住建领域

住建领域知识的特点是**强关联结构**：

```
法规A → 包含 → 第X条 → 引用 → 标准B → 触发 → 处罚C → 适用于 → 案例D
```

## 二、架构设计

### 2.1 整体流程

```
用户提问："噪声超标怎么处罚？"
        │
        ▼
┌─────────────────────────────────┐
│  第一步：向量检索（粗筛）         │
│  问"噪声超标处罚" → 找出相关文档  │
└─────────────────┬───────────────┘
                  │
                  ▼
┌─────────────────────────────────┐
│  第二步：Neo4j 图谱推理（深挖）   │
│  查这些文档之间的关联关系        │
│  法规→条款→标准→处罚→案例       │
└─────────────────┬───────────────┘
                  │
                  ▼
┌─────────────────────────────────┐
│  第三步：LLM 生成回答             │
│  综合内容整理成流畅的回答        │
└─────────────────────────────────┘
```

### 2.2 数据同步流程

```
知识入库
   │
   ├──────────────────────────────────────┐
   ▼                                      ▼
┌─────────────────┐              ┌─────────────────┐
│  存入 MySQL/ES  │              │  异步触发抽取   │
│  (原有流程)      │              │  graph_extractor │
└─────────────────┘              └────────┬────────┘
                                           │
                    ┌──────────────────────┼──────────────────────┐
                    ▼                      ▼                      ▼
             ┌───────────┐         ┌───────────┐         ┌───────────┐
             │ 规则抽取  │         │ LLM 抽取  │         │ 关系补全  │
             │ (快速)    │         │ (语义)    │         │ (补充)    │
             └─────┬─────┘         └─────┬─────┘         └─────┬─────┘
                   └──────────────────────┼──────────────────────┘
                                           ▼
                                  ┌─────────────────┐
                                  │   写入 Neo4j    │
                                  │  (MERGE 节点+关系)│
                                  └─────────────────┘
```

## 三、实体类型定义

### 3.1 实体类型表

| 实体类型 | Label | 说明 | 示例 |
|---------|-------|------|------|
| 法规 | `:Law` | 法律法规名称 | 《建设工程施工现场噪声污染防治法》 |
| 条款 | `:Article` | 具体法条 | 第2条、第3条第1款 |
| 标准 | `:Standard` | 技术标准 | GB 3096-2008 |
| 处罚 | `:Penalty` | 处罚内容 | 罚款5000-50000元、责令停产停业 |
| 案例 | `:Case` | 案例 | XX项目噪声超标处罚案 |
| 主体 | `:Subject` | 当事人 | XX建筑工程公司 |
| 行为 | `:Behavior` | 违规行为 | 噪声超标排放 |

### 3.2 节点属性

#### Law（法规）

```cypher
(:Law {
  name: String,           // 法规名称（唯一标识）
  code: String,           // 法规编号
  source: String,         // 知识库ID
  category: String,       // law/tech/case
  created_at: DateTime
})
```

#### Article（条款）

```cypher
(:Article {
  name: String,           // 条款编号（唯一标识）
  content: String,        // 条款原文摘要
  law_name: String,       // 所属法规名称
  article_no: String      // 条款号
})
```

## 四、关系类型定义

### 4.1 关系类型表

| 关系 | Type | 说明 | 示例 |
|------|------|------|------|
| 包含 | `:CONTAINS` | 法规包含条款 | (噪声法)-[:CONTAINS]->(第2条) |
| 引用 | `:REFERS` | 条款引用标准 | (第2条)-[:REFERS]->(GB3096) |
| 规定 | `:DEFINES` | 条款规定行为 | (第2条)-[:DEFINES]->(噪声超标) |
| 触发 | `:TRIGGERS` | 行为触发处罚 | (噪声超标)-[:TRIGGERS]->(罚款) |
| 适用 | `:APPLIES` | 案例适用条款 | (XX案)-[:APPLIES]->(第2条) |
| 涉及 | `:INVOLVES` | 案例涉及主体 | (XX案)-[:INVOLVES]->(施工单位) |
| 施加 | `:IMPOSES` | 处罚施加主体 | (罚款)-[:IMPOSES]->(施工单位) |
| **抽取自** | **:EXTRACTED_FROM** | **实体来源** | **(实体)-[:EXTRACTED_FROM]->(知识)** |

## 六、知识库同步钩子

### 6.1 新增知识时

在 `knowledge.py` 的 `upload_knowledge` 和 `create_knowledge_manual` 中添加：

```python
# 异步同步到 Neo4j（图谱抽取，不阻塞主流程）
if parsed_content:
    asyncio.create_task(
        asyncio.to_thread(_sync_knowledge_to_neo4j, knowledge_id, parsed_content)
    )
```

### 6.2 删除知识时

```python
# 删除 Neo4j 相关数据
extractor.delete_from_neo4j(knowledge_id)
```

### 6.3 删除逻辑

```
知识删除时
    │
    ▼
┌─────────────────────────────────┐
│  查找该知识关联的所有实体        │
│  (通过 EXTRACTED_FROM 关系)     │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────────────────────────┐
│  对每个实体：                    │
│  - 移除 EXTRACTED_FROM 关系     │
│  - 检查 ref_count                │
│    - ref_count == 0 → 删除实体   │
│    - ref_count > 0 → 保留       │
└─────────────────────────────────┘
```
