# 智能助手-图谱增强问答-图谱上下文增强-详细设计

> 最后更新：2026-04-28

## 一、问题背景

### 1.1 原有问题

在 `reason_with_neo4j` 方法中，Neo4j 图谱数据只用于前端可视化（`graph_data`），没有参与 LLM 回答生成：

```python
# 原代码
nodes, edges = self._build_graph_from_neo4j(neo4j_context, question)
graph_data = {"nodes": nodes, "edges": edges}  # 只给前端

final_answer = self._generate_answer(question, unique_items)  # 不包含图谱数据！
```

**问题**：用户看到漂亮的图谱，但回答质量没有提升。

### 1.2 解决方案

将 Neo4j 图谱关系数据真正加入 LLM 的 prompt 中，让 LLM 知道知识点之间的"关系和顺序"。

## 二、代码改动

### 2.1 新增 `_build_graph_context` 方法

```python
def _build_graph_context(self, neo4j_context: Dict[str, Any], entities: List[Dict]) -> str:
    """
    从 Neo4j 上下文构建可读的图谱描述，用于补充 LLM 回答

    Args:
        neo4j_context: Neo4j 查询结果（包含 nodes, edges）
        entities: 原始识别的实体列表

    Returns:
        可读性好、可直接加入 prompt 的图谱描述字符串
    """
```

**输出格式：**
```
=== 知识图谱关系（供参考）===

【实体分类】
  法规：《安全生产许可证条例》, 《建筑法》, ...
  条款：第一条, 第二条, 第三条
  主体：住房公积金所有者

【关系链条】
  • 《安全生产许可证条例》 包含 第一条
  • 第一条 包含 《建筑法》
  • ...

（以上图谱包含 69 个实体和 97 条关系）
```

### 2.2 修改 `_generate_answer` 方法

新增 `graph_context` 参数：

```python
def _generate_answer(self, question: str, search_results: List[Dict], graph_context: str = "") -> str:
```

在 prompt 中加入图谱上下文：

```python
if graph_context:
    prompt += f"\n{graph_context}\n"
```

### 2.3 新增书名号匹配方法

```python
def _match_neo4j_entity(self, entity_name: str, neo4j) -> Optional[str]:
    """
    匹配 Neo4j 中的实体名称（处理书名号差异）

    - "安全生产许可证条例" → 匹配 → "《安全生产许可证条例》"
    - "《建筑法》" → 匹配 → "建筑法"（如果没找到）
    """
```

## 三、合并流程对比

### 3.1 重构前

```
Step 2 向量搜索 → all_search_items
Step 3 Neo4j 图谱 → graph_data（只给前端）
Step 4 去重 → unique_items
Step 5 生成回答 → _generate_answer(question, unique_items)  # 无图谱数据
```

### 3.2 重构后

```
Step 2 向量搜索 → all_search_items
Step 3 Neo4j 图谱 → graph_data（给前端）+ neo4j_context（传给 LLM）
Step 4 构建图谱上下文 → graph_context = _build_graph_context(neo4j_context, entities)
Step 5 去重 → unique_items
Step 6 生成回答 → _generate_answer(question, unique_items, graph_context)  # 有图谱数据
```

## 四、最终传给 LLM 的数据结构

### 4.1 Prompt 结构

```
你是住建领域知识助手。请根据以下知识内容回答用户问题。

用户问题：{question}

知识内容：
【知识 1】{title1}
{content1[:500]}

【知识 2】{title2}
{content2[:500]}

...

=== 知识图谱关系（供参考）===

【实体分类】
  法规：...
  条款：...

【关系链条】
  • 实体A 包含 实体B
  • ...

（以上图谱包含 X 个实体和 Y 条关系）

请给出专业、准确的回答。如果知识不足以回答，请说明。
回答格式：先给出核心答案，然后详细说明依据。
```

### 4.2 示例数据

**问题：** "安全生产许可证条例怎么办理"

| 数据来源 | 内容 |
|---------|------|
| 知识内容 | 建筑施工企业安全生产许可证管理规定、建筑施工特种作业人员管理规定... |
| 图谱实体 | 《安全生产许可证条例》(Law), 第一条(Article), 《建筑法》(Law)... |
| 图谱关系 | 《安全生产许可证条例》 包含 第一条, 第一条 包含 《建筑法》... |

## 五、效果对比

| 对比项 | 重构前 | 重构后 |
|--------|--------|--------|
| LLM 知道的内容 | 零散的知识文章 | 知识文章 + 知识点之间的关系 |
| 回答内容 | "根据相关规定..." | "根据图谱关系，流程是：1→2→3..." |
| 流程顺序 | 模糊 | 清晰（1→2→3） |
| 部门归属 | 不明确 | 明确（如：住建局监管） |
