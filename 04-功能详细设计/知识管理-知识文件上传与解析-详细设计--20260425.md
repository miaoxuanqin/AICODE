# 知识文件上传与解析详细设计

> 创建时间：2026-04-25
> 更新时间：2026-04-25

## 一、功能目标

实现用户上传 Word/PDF 文件，自动解析提取文本内容，建立知识索引。文件存储到 MinIO，**解析后content存入Elasticsearch**，MySQL只存元数据和摘要。

## 二、接口说明

### 2.1 文件上传接口

**POST** `/api/v1/knowledge/upload`

**请求**
- Content-Type: `multipart/form-data`
- Body:
  - `file`: 文件（必填）
  - `title`: 知识标题（可选，默认用文件名）
  - `category`: 分类（必填，law/tech/case/policy）
  - `source`: 来源（可选）
  - `tags`: 标签，逗号分隔（可选）

**响应**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "uuid",
    "title": "建设工程质量管理条例",
    "category": "law",
    "file_type": "pdf",
    "status": "parsed"
  }
}
```

**业务规则**
1. 文件大小限制：50MB
2. 支持格式：.doc, .docx, .pdf
3. 解析失败时，知识状态标记为 `parse_failed`，保留文件但不可搜索

## 三、文件解析流程

```
1. 接收文件流
2. 生成 UUID 作为知识ID
3. 上传到 MinIO bucket: knowledge, path: {user_id}/{uuid}.{ext}
4. 根据文件类型选择解析器：
   - PDF → pdfplumber 提取文本
   - Word (.docx/.doc) → python-docx 提取文本（含chunk.xhtml兼容）
   - Word (.doc) → Windows COM自动化（如果可用）
5. 提取内容：
   - title: 取文件标题或文档第一行
   - content: 完整文本内容
   - summary: 前 500 字作为摘要
6. MySQL只存：title, summary, category, source, tags, file_path
7. Elasticsearch存储：完整content（全文搜索）
8. Qdrant向量：待接入embedding模型
```

## 四、数据存储

### 4.1 MinIO 存储

| 字段 | 说明 |
|-----|------|
| Bucket | knowledge |
| Path | {user_id}/{knowledge_id}.{ext} |
| 元数据 | Content-Type, x-amz-meta-user_id |

### 4.2 MySQL 存储

| 字段 | 说明 |
|-----|------|
| id | UUID主键 |
| title | 标题 |
| summary | 摘要（前500字） |
| content | **已移除，存ES** |
| category | 分类 |
| source | 来源 |
| tags | 标签JSON |
| file_path | MinIO文件路径 |
| file_type | 文件类型 |

### 4.3 Elasticsearch 索引

**索引名**: `knowledge`

**Mapping**:
```json
{
  "mappings": {
    "properties": {
      "id": { "type": "keyword" },
      "title": { "type": "text" },
      "content": { "type": "text" },
      "summary": { "type": "text" },
      "category": { "type": "keyword" },
      "source": { "type": "text" },
      "tags": { "type": "keyword" },
      "user_id": { "type": "keyword" },
      "created_at": { "type": "date" },
      "view_count": { "type": "integer" }
    }
  }
}
```

**注意**: 使用默认分词器，未安装ik_max_word插件

### 4.4 Qdrant 索引

**Collection**: `knowledge`
**Vector Size**: 1536
**状态**: 待接入embedding模型

## 五、异常处理

| 场景 | 处理方式 |
|-----|---------|
| 文件格式不支持 | 返回 400，提示支持的格式 |
| 文件超过 50MB | 返回 413，提示文件过大 |
| 解析失败 | 状态设为 parse_failed，记录错误日志 |
| MinIO 连接失败 | 返回 500，提示存储服务异常 |
| ES 索引失败 | 入库成功但搜索不可用，异步重试 |
| Qdrant 索引失败 | 降级为纯关键词搜索 |

## 六、验收标准

1. 上传 Word/PDF 文件，解析入库成功
2. 解析后知识可立即被搜索到
3. 不同用户的知识数据完全隔离
4. 详情页能显示完整content
