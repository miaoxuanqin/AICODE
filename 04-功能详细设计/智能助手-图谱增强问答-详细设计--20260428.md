## 三、功能详情

### 1. 对话功能

#### 1.1 消息结构

```javascript
// 用户消息
{
  role: 'user',
  content: '工地脚手架坠落事故如何处罚？',
  time: '2026-04-27 10:30:00'
}

// 助手消息
{
  role: 'assistant',
  content: '## 事故分析与处罚建议\n\n...',  // Markdown 格式
  reasoningChain: [  // 推理链
    {
      step: 1,
      query: '理解问题并识别关键实体',
      result: '识别到 5 个关键实体',
      entities: ['脚手架', '坠落', '事故', '安全', '生产'],
      details: {
        search_query: '工地脚手架坠落事故如何处罚',
        knowledge_found: []
      }
    },
    {
      step: 2,
      query: '检索相关知识库内容',
      result: '找到 6 条相关知识',
      entities: [],
      details: {
        search_query: '工地脚手架坠落事故如何处罚',
        knowledge_found: ['建筑施工特种作业人员管理规定', '建设工程安全生产管理条例', ...]
      }
    }
  ],
  citations: [  // 引用来源
    { id: 'n1', type: 'law', typeName: '法规', title: '建设工程安全生产管理条例' }
  ],
  time: '2026-04-27 10:30:05'
}
```

## 四、前端组件状态

### 4.1 状态变量

```javascript
// 对话相关
const inputMessage = ref('')           // 输入内容
const isThinking = ref(false)          // 加载状态
const messages = ref([])               // 消息列表

// 推理相关
const showReasoning = ref(true)       // 显示推理过程
const thinkingSteps = ['理解问题', '检索知识', '多跳推理', '生成回答']
const thinkingCurrent = ref(-1)
const expandedSteps = ref([])         // 展开的步骤索引

// 图谱相关
const graphData = reactive({ nodes: [], edges: [] })
const selectedNode = ref(null)         // 选中节点
const hoveredNodeId = ref(null)        // 悬停节点ID
const highlightedNodes = ref([])
const highlightedEdges = ref([])
```

### 4.2 核心方法

```javascript
// 切换推理步骤展开
const toggleStep = (stepIdx) => {
  const idx = expandedSteps.value.indexOf(stepIdx)
  if (idx > -1) {
    expandedSteps.value.splice(idx, 1)
  } else {
    expandedSteps.value.push(stepIdx)
  }
}

// 发送消息
const handleSend = async () => {
  const result = await graphApi.qa(userMsg.content)

  if (result.graph_data) {
    graphData.nodes = result.graph_data.nodes || []
    graphData.edges = result.graph_data.edges || []
  }

  const aiMsg = {
    role: 'assistant',
    content: result.answer,
    reasoningChain: result.reasoning_chain || [],
    citations: result.citations || [],
    time: new Date().toLocaleTimeString()
  }
  messages.value.push(aiMsg)
}

// 选择节点
const selectNode = (node) => {
  selectedNode.value = node
  highlightedNodes.value = [node.id]
  const relatedEdges = graphData.edges.filter(e => e.source === node.id || e.target === node.id)
  highlightedEdges.value = relatedEdges.map(e => e.id)
}
```

## 七、Bug修复记录

### 2026-04-27 晚

| Bug | 原因 | 修复方案 |
|-----|------|----------|
| 图谱节点悬停乱跳 | `hoveredNode` 存储响应式对象导致重渲染 | 改用 `hoveredNodeId` 存储 ID |
| 图谱节点悬停乱跳 | CSS `transform: scale(1.1)` 与 SVG `translate()` 冲突 | 移除问题 CSS |
| 推理步骤详情无法查看 | Pydantic `ReasoningStep` 缺少 `details` 字段被过滤 | 添加 `details: Dict[str, Any]` 字段 |
| 图谱问答 404 | Python `__pycache__` 缓存旧字节码 | 清除缓存后重启 |

## 九、Neo4j 集成规划

> 以下为 2026-04-28 新增内容

### 9.1 集成方案概述

**推荐方案**：向量检索 + Neo4j 图数据库结合使用

| 组件 | 作用 |
|------|------|
| 向量检索 | 快速找到"可能相关"的知识（广撒网） |
| Neo4j | 理清知识之间的"关系链"（深挖掘） |
| LLM | 整理成人类可读的回答 |

### 9.2 架构流程

```
用户提问
   │
   ▼
┌─────────────────────────────────┐
│  第一步：向量检索                │
│  问"噪声超标处罚" → 找出相关文档  │
│  《噪声法》《处罚案例》等         │
└─────────────────┬───────────────┘
                  │
                  ▼
┌─────────────────────────────────┐
│  第二步：Neo4j 图谱推理          │
│  查这些文档之间的关联关系        │
│  法规→条款→标准→处罚→案例       │
└─────────────────┬───────────────┘
                  │
                  ▼
┌─────────────────────────────────┐
│  第三步：LLM 生成回答             │
│  综合内容整理成流畅的回答        │
└─────────────────────────────────┘
```

### 9.3 实体类型定义

| 实体类型 | 说明 | 示例 |
|---------|------|------|
| `Law` | 法规名称 | 《建设工程施工现场噪声污染防治法》 |
| `Article` | 具体条款 | 第X条第Y款 |
| `Standard` | 技术标准 | GB 3096-2008、GB 50189 |
| `Penalty` | 处罚内容 | 罚款5000-50000元、责令停产停业 |
| `Case` | 案例 | XX项目噪声超标处罚案 |
| `Subject` | 主体 | 建设单位、施工单位、监管部门 |
| `Behavior` | 行为 | 噪声超标排放、违规夜间施工 |

### 9.4 关系类型定义

| 关系 | 说明 | 示例 |
|------|------|------|
| `CONTAINS` | 法规包含条款 | 噪声法 → 包含 → 第X条 |
| `REFERS` | 条款引用标准 | 第X条 → 引用 → GB 3096 |
| `DEFINES` | 条款规定行为 | 第X条 → 规定 → 噪声排放 |
| `TRIGGERS` | 行为触发处罚 | 噪声超标 → 触发 → 罚款 |
| `APPLIES` | 案例适用条款 | XX案 → 适用 → 第X条 |
| `INVOLVES` | 案例涉及主体 | XX案 → 涉及 → 施工单位A |
| `IMPOSES` | 处罚施加于主体 | 罚款2万 → 施加于 → 施工单位A |
