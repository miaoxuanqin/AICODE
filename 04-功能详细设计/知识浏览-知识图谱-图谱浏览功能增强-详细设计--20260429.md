##### 3. 发现相似按钮
- **功能**：搜索相似名称的节点并加载到图谱
- **实现**：
```javascript
const findSimilarNodes = async () => {
  if (!selectedNode.value) return
  loading.value = true
  try {
    const res = await graphApi.searchNodes(selectedNode.value.name, null, 10)
    if (res.results?.length > 0) {
      const similar = res.results.filter(r => r.id !== selectedNode.value.id).slice(0, 5)
      if (similar.length > 0) {
        await loadResultsToGraph(similar)
        ElMessage.success(`已加载 ${similar.length} 个相似实体`)
      }
    }
  } finally {
    loading.value = false
  }
}
```

##### 5. 加载相关推荐
```javascript
const loadRelatedRecommendations = async () => {
  if (!selectedNode.value) return
  try {
    const res = await graphApi.neighbors(selectedNode.value.name, 2)
    if (res.nodes) {
      const recommendations = res.nodes
        .filter(n => n.id !== selectedNode.value.id)
        .slice(0, 5)
        .map(n => ({ id: n.id, name: n.name, label: n.label }))
      relatedRecommendations.value = recommendations
    }
  } catch (error) {
    console.error('加载相关推荐失败:', error)
  }
}
```

### 2.2 图例点击功能增强

#### 原行为
点击图例只切换类型筛选（高亮/隐藏该类型节点）

#### 新行为
点击图例时：
- 如果图谱为空或该类型节点未加载，自动加载该类型的前20个节点
- 如果已有该类型节点，执行筛选切换

#### 实现
```javascript
const handleLegendClick = async (typeLabel) => {
  console.log('handleLegendClick 被调用, typeLabel:', typeLabel)
  ElMessage.info(`正在加载${typeLabel}类型节点...`)
  await toggleTypeFilter(typeLabel)
}

const toggleTypeFilter = async (type) => {
  // 直接加载该类型的前20个节点
  try {
    const res = await graphApi.searchNodes('', type, 20)
    if (res.results?.length > 0) {
      await loadResultsToGraph(res.results)
      typeFilter.value = [type]
      return
    } else {
      ElMessage.warning('暂无数据')
      return
    }
  } catch (e) {
    console.error('加载类型节点失败:', e)
    ElMessage.error('加载失败')
  }
}
```

### 2.3 边交互功能

#### 点击边高亮路径
- **触发**：点击边
- **效果**：
  1. 被点击的边变粗、变蓝色
  2. 边两端的节点边框加粗
  3. 2秒后自动恢复

#### 实现
```javascript
network.on('click', (params) => {
  if (params.nodes.length > 0) {
    // 节点点击处理
  } else if (params.edges.length > 0) {
    highlightEdgePath(params.edges[0])
  }
})

const highlightEdgePath = (edgeId) => {
  if (!network) return
  const edge = edges.value.find(e => e.id === edgeId)
  if (!edge) return

  const connectedNodeIds = [edge.from, edge.to]

  // 高亮边
  network.body.data.edges.update({
    id: edgeId,
    color: { color: '#409eff', highlight: '#67c23a' },
    width: 3
  })

  // 高亮节点
  connectedNodeIds.forEach(nodeId => {
    network.body.data.nodes.update({
      id: nodeId,
      borderWidth: 3
    })
  })

  // 2秒后恢复
  setTimeout(() => {
    // 重置样式
  }, 2000)
}
```

## 四、Bug 修复

### 4.1 handleRelationClick 重复定义

- **问题**：编译错误 "The symbol handleRelationClick has already been declared"
- **原因**：函数在两个位置被定义
- **修复**：删除重复定义，保留一个实现

### 4.2 节点名称显示为关系类型

- **问题**：节点显示的名称是关系类型名称（如 CONTAINS、REFERS）
- **原因**：renderNetwork 中使用了错误的字段
- **修复**：
```javascript
label: (n.name || n.label || '').length > 15 ? ... : (n.name || n.label || ''),
```

### 4.3 节点字段映射问题

- **问题**：选择节点时提示"节点名称为空"
- **原因**：后端返回的字段名与前端期望的不一致
- **修复**：
  - 前端 `loadResultsToGraph` 中正确映射 name、label、title、type
  - 统一使用 `node.name` 或 `node.label` 作为节点名称

### 4.4 边字段映射问题

- **问题**：边没有正确显示
- **原因**：`neighbors` API 返回 `source`/`target`，前端使用 `from`/`to`
- **修复**：统一使用 `source`/`target`
