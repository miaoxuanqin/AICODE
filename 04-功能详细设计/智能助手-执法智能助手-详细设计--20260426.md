## 一、技术架构

```
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│   前端页面           │     │   FastAPI           │     │   MiniMax LLM       │

                                        │
                               ┌────────▼────────┐
                               │  Elasticsearch │
                               │   关键词搜索    │
                               └─────────────────┘
```

### 核心流程

1. 用户输入场景描述（或选择场景模板）
2. 调用 `qaApi.chat(question, userId, sessionId)`
3. 后端：ES 搜索 → 构建 Prompt → LLM 生成回答
4. 前端：展示回答 + 知识卡片 + 相关问题推荐

### 3.4 handleSend 实现

```javascript
const handleSend = async () => {
  if (!inputMessage.value.trim()) return

  const userMsg = {
    role: 'user',
    content: inputMessage.value,
    time: new Date().toLocaleTimeString()
  }
  messages.value.push(userMsg)
  inputMessage.value = ''
  isThinking.value = true

  scrollToBottom()

  try {
    const userId = cachedUserId || await ensureUserId()
    const result = await qaApi.chat(userMsg.content, userId, currentSessionId.value)

    const aiMsg = {
      role: 'assistant',
      content: result.answer,
      cards: result.cards || [],
      relatedQuestions: result.related_questions || [],
      time: new Date().toLocaleTimeString()
    }
    messages.value.push(aiMsg)

    if (result.session_id && result.session_id !== currentSessionId.value) {
      currentSessionId.value = result.session_id
    }
  } catch (error) {
    ElMessage.error('问答服务出错，请稍后重试')
  } finally {
    isThinking.value = false
  }

  scrollToBottom()
}
```

## 四、会话管理策略

### 4.1 设计原则

- **静默管理** - 用户无感知，不显示会话列表
- **自动关联** - 进入页面时自动创建/获取会话
- **复用服务** - 完全复用 qa_service 的 Session 管理
- **会话隔离** - 通过 category 字段区分不同助手

### 4.2 category 隔离方案

| category | 用途 | 可见页面 |
|----------|------|----------|
| qa | 问答助手会话 | QAChat.vue |
| law_general | 执法助手会话 | LawAssistant.vue |

### 4.3 实现方式

```javascript
// 进入页面时初始化会话
const initSession = async () => {
  try {
    const data = await qaApi.createSession('【执法】新对话', 'law_general')
    currentSessionId.value = data.id
  } catch (error) {
    console.error('创建执法会话失败:', error)
  }
}

onMounted(async () => {
  await ensureUserId()
  await initSession()
})
```

### 4.4 API 调用链

```
用户发送消息
    ↓
qaApi.chat(question, userId, sessionId)
    ↓
后端：qa_service.ask()
    ↓
1. 关联/创建 Session (category='law_general')
2. 保存用户消息到 Session
3. ES 搜索
4. LLM 生成回答
5. 保存助手消息到 Session
6. 返回结果（含 session_id）
    ↓
前端：更新 currentSessionId
```

## 六、后端改动

### 6.1 QASession 模型

```python
class QASession(Base):
    __tablename__ = "qa_session"

    id = Column(String(36), primary_key=True)
    user_id = Column(String(36), nullable=False, index=True)
    title = Column(String(200), default='新对话')
    category = Column(String(50), default='qa', index=True)  # qa/law_general/supervise
    is_active = Column(SmallInteger, default=1)
    created_at = Column(DateTime, server_default=func.now())
    updated_at = Column(DateTime, server_default=func.now(), onupdate=func.now())
```

### 6.2 API 改动

**POST /api/v1/qa/session** - 支持 title 和 category 参数

```python
@router.post("/session", response_model=QASessionResponse)
async def create_session(
    request: QASessionCreateRequest = None,
    db: Session = Depends(get_db),
    current_user = Depends(get_current_user)
):
    title = request.title if request else "新对话"
    category = request.category if request else "qa"
    session = qa_service.create_session(
        str(current_user.id), db,
        title=title, category=category
    )
    return session
```

### 6.3 数据库迁移

```sql
-- 15-qa-session-category.sql

ADD COLUMN category VARCHAR(50) DEFAULT 'qa' AFTER title;

ALTER TABLE qa_session
ADD INDEX idx_category (category);
```

## 八、异常处理

| 场景 | 处理 |
|------|------|
| 网络错误 | ElMessage.error 提示 |
| LLM 调用失败 | 返回"服务暂时不可用" |
| ES 无结果 | 返回"知识库中暂无相关内容" |
| Session 失效 | 重新创建会话 |
