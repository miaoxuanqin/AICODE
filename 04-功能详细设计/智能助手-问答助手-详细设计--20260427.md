## 一、技术架构

### 1.1 整体架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端页面   │────▶│   FastAPI   │────▶│     LLM     │
│  QAChat.vue │     │  /api/v1/qa │     │ MiniMax-M2.7│
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                     ┌─────▼─────┐
                     │  Embedding │
                     │  → Qdrant  │
                     └───────────┘
```

### 1.2 核心流程（2026-04-27 更新）

1. **用户提问** → 前端发送问题到后端
2. **向量检索** → Embedding 模型编码 → Qdrant 向量搜索
3. **获取知识详情** → Qdrant 返回 ID → ES 获取完整信息
4. **构建 Prompt** → 将问题和知识内容组合（格式优化，不再显示【知识1】）
5. **LLM 生成** → 调用 MiniMax 模型（正确处理 ThinkingBlock + TextBlock）
6. **返回结果** → 前端展示答案和知识卡片

### 1.3 向量搜索流程

```
问题 → Embedding.encode_single() → Qdrant.search() → 知识ID列表
                                              ↓
                                      ES.get_by_id() → 知识详情
```

### 2.4 Prompt 模板（2026-04-27 更新）

**知识内容格式**：从 `【知识 1】标题...内容` 改为 `[1] 标题\n分类：xxx\n内容`

```python
def _build_prompt(self, question: str, search_results: List[Dict],
                  history_messages: List[Dict] = None,
                  category: str = "qa") -> str:
    """构建 Prompt（支持对话历史和 category 差异化）"""
    config = self.CATEGORY_CONFIG.get(category, self.CATEGORY_CONFIG["qa"])
    system_prompt = config["system_prompt"]

    # 对话历史
    history_text = ""
    if history_messages:
        history_parts = []
        for msg in history_messages:
            role = "用户" if msg['role'] == 'user' else "助手"
            history_parts.append(f"{role}：{msg['content']}")
        history_text = "\n".join(history_parts)

    # 知识内容（格式优化，不再显示【知识1】等）
    if not search_results:
        knowledge_text = "（知识库中暂无相关内容）"
    else:
        knowledge_parts = []
        for i, item in enumerate(search_results, 1):
            content = item.get('summary', '') or item.get('content', '')
            knowledge_parts.append(
                f"[{i}] {item['title']}\n分类：{item.get('category', '通用')}\n{content[:2000]}"
            )
        knowledge_text = "\n\n".join(knowledge_parts)

    if history_text:
        return f"""{system_prompt}

【参考信息】
{knowledge_text}

【当前问题】
{question}

请根据对话历史和参考信息给出准确、专业的回答。如果参考信息不足以回答，请说明。"""
    else:
        return f"""{system_prompt}

【参考信息】
{knowledge_text}

【用户问题】
{question}

请根据参考信息给出准确、专业的回答。如果参考信息不足以回答，请说明。"""
```

### 2.5 LLM 调用（2026-04-27 修复）

def _call_llm(self, prompt: str) -> str:

    try:
        message = self.llm_client.messages.create(
            model=settings.anthropic_default_haiku_model,

### 3.2 会话管理接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建会话 | POST | /api/v1/qa/session | 可指定 title 和 category |
| 获取会话列表 | GET | /api/v1/qa/sessions | 返回用户所有会话 |
| 获取会话详情 | GET | /api/v1/qa/session/{id} | 返回会话消息历史 |

## 四、数据模型

### 4.1 会话表 (qa_session)

```sql
CREATE TABLE qa_session (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    title VARCHAR(200) DEFAULT '新对话',
    category VARCHAR(50) DEFAULT 'qa',  -- qa/law_general/supervise
    is_active INT DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_category (category)
);
```

### 4.2 会话消息表 (qa_session_message)

```sql
CREATE TABLE qa_session_message (
    id VARCHAR(50) PRIMARY KEY,  -- 格式: {timestamp}-user/assistant
    session_id VARCHAR(36) NOT NULL,
    role VARCHAR(20) NOT NULL,  -- user/assistant
    content TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES qa_session(id) ON DELETE CASCADE
);
```
