## 3. 数据存储架构

### 3.1 存储位置说明

| 存储位置 | 存储内容 | 说明 |
|---------|---------|------|
| **MySQL** | 元数据 | `id`, `title`, `type`, `category`, `source`, `tags`, `file_path`, `created_at` 等（不存内容） |
| **ES (Elasticsearch)** | 内容 | `doc` 类型：文件解析后的文本；`text` 类型：原始文本 |
| **向量库** | 向量化数据 | 文本向量化 |
| **知识图谱库** | 实体和关系 | 从文本中提取 |
| **MinIO** | 原始文件 | 仅 `doc`/`pdf` 类型 |

### 3.2 知识类型

| 类型 | 值 | 说明 | 存储内容 |
|------|-----|------|---------|
| PDF文档 | `pdf` | PDF格式文件 | MinIO存原文件，ES存解析文本 |
| Word文档 | `doc` | Word格式文件 | MinIO存原文件，ES存解析文本 |
| 文本 | `text` | 纯文本内容 | ES存原始文本 |

## 4. 知识字段定义

### 4.1 MySQL字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 知识ID |
| `title` | string | 标题 |
| `type` | enum | `pdf`/`doc`/`text` |
| `category` | enum | `law`(法律法规)/`tech`(技术标准)/`case`(执法案例)/`policy`(政策文件) |
| `source` | string | 来源，如"国务院令第279号" |
| `tags` | array | 标签数组 |
| `file_path` | string | MinIO文件路径（仅文档类型） |
| `created_at` | datetime | 创建时间 |

### 4.2 状态字段

| 字段 | 说明 | 值 |
|------|------|-----|
| `fullTextStatus` | 全文检索状态 | `indexed`（已索引）/`pending`（待索引） |
| `vectorStatus` | 语义搜索状态 | `done`（已完成）/`pending`（待处理） |
| `graphStatus` | 知识图谱状态 | `done`（已完成）/`pending`（待处理） |

## 5. 功能设计

### 5.1 文件上传

**入口**：点击「上传文档」按钮

**表单字段**：
| 字段 | 必填 | 说明 |
|------|------|------|
| 文件 | ✅ | 支持拖拽或点击上传，`.pdf`、`.doc`、`.docx`，最大 50MB |
| 标题 | 否 | 不填则使用文件名 |
| 分类 | ✅ | law/tech/case/policy |
| 来源 | 否 | 如"国务院令第279号" |
| 标签 | 否 | 多个标签用逗号分隔 |

**后端处理流程**：
1. 文件存 MinIO
2. 解析文件内容（PDF/Word → 文本）
3. 存入 ES
4. 生成向量存向量库
5. 提取实体关系存入知识图谱
6. MySQL 只存元数据

### 5.2 手动添加文本

**入口**：点击「添加文本」按钮

**表单字段**：
| 字段 | 必填 | 说明 |
|------|------|------|
| 标题 | ✅ | 知识标题 |
| 内容 | ✅ | 纯文本内容 |
| 分类 | ✅ | law/tech/case/policy |
| 来源 | 否 | |
| 标签 | 否 | |

**后端处理流程**：
1. 直接存原始文本到 ES
2. 生成向量存向量库
3. 提取实体关系存知识图谱
4. MySQL 只存元数据

### 5.3 删除知识

删除时需清理所有关联数据：

| 类型 | 清理项 |
|------|--------|
| **所有类型** | MySQL 元数据、ES 内容、向量库数据、知识图谱数据 |
| **文档类型** (`doc`/`pdf`) | 额外清理 MinIO 存储文件 |

## 6. 视图模式

### 6.1 列表视图

- 横向排列知识卡片
- 左侧：类型图标、标题、分类、来源、时间
- 中间：三个状态卡片（全文检索/语义搜索/知识图谱）
- 右侧：操作按钮（查看/编辑/删除）

### 6.2 网格视图

- 卡片式布局（自适应列数）
- 顶部：标题、类型图标
- 中部：分类、来源、时间元信息
- 下部：三个状态卡片
- 底部：操作按钮

### 6.3 状态管理

每个状态卡片支持：
- **重建**：重新执行对应处理流程
- **清空**：清除对应库中的数据
