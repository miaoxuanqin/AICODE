# 手动添加知识功能详细设计

> 创建时间：2026-04-26

## 需求背景

原知识管理只支持上传文件（PDF/Word）方式创建知识，系统自动解析提取内容。现需要增加**手动编辑**方式：
- 自己输入知识标题
- 自己编辑知识内容（支持富文本格式）
- 选择分类、来源、标签等

## 功能设计

### 后端接口

**新增接口：`POST /api/v1/knowledge/manual`**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 知识标题 |
| content | string | 是 | 富文本内容（HTML 格式） |
| category | string | 是 | 分类：law/tech/case/policy |
| source | string | 否 | 来源 |
| tags | string[] | 否 | 标签数组 |

**响应：**
```json
{
  "id": "知识ID",
  "title": "标题",
  "category": "law",
  "file_type": "html",
  "status": "active"
}
```

### 数据存储

- **MySQL**：title, summary, content, category, source, tags, user_id, status 等
- **Elasticsearch**：content 完整索引（支持全文搜索）

### 前端页面

**KnowledgeManage.vue 改动：**

1. 标题栏新增"手动添加"按钮（绿色）
2. 弹窗包含：
   - 标题输入框
   - 富文本编辑器（vue-quill）
   - 分类下拉选择
   - 来源输入框
   - 标签输入框

### 富文本编辑器选型

| 方案 | 评估 |
|------|------|
| wangEditor V5 | Vue3 版本 peer dependency 不兼容，导入方式复杂 |
| @vueup/vue-quill | Vue3 官方封装 Quill，支持完整工具栏，推荐 |

**最终选用：@vueup/vue-quill**

### 工具栏配置

```javascript
toolbar="full"
// 支持：加粗、斜体、下划线、删除线、标题、引用、列表、代码块、链接、图片等
```

## 页面交互

1. 点击"手动添加"按钮 → 弹出对话框
2. 填写标题、内容、选择分类
3. 点击"保存" → 调用 POST /knowledge/manual
4. 保存成功后关闭对话框，刷新知识列表

## 已知问题修复

### 1. Quill placeholder 竖排问题

**问题**：placeholder 文字"请输入知识内容..."显示为竖排。

**原因**：Element Plus 样式与 Quill 默认样式冲突。

**解决方案**：在组件样式中添加：
```css
:deep(.ql-editor.ql-blank::before) {
  writing-mode: horizontal-tb !important;
  color: #999;
  font-style: normal;
}
```

### 2. 编辑器布局错乱

**问题**：富文本框占据分类、来源、标签的位置。

**原因**：Quill 编辑器样式未正确包裹。

**解决方案**：使用 wrapper div 包裹编辑器，设置明确高度：
```css
.editor-wrapper {
  border: 1px solid #dcdfe6;
  border-radius: 4px;
  overflow: hidden;
}
```
